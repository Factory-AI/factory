# IA map

The canonical IA lives in `docs/docs.json`; inspect it instead of relying on a
copied navigation map.

## Placement rules

- Put setup and first-run material near existing onboarding pages.
- Put reusable workflows near existing guide pages.
- Put shipped settings, flags, schemas, and CLI options near existing reference
  or configuration pages.
- Put admin controls, security, governance, deployment, and analytics near
  existing enterprise pages.
- Keep hidden reusable snippets under `docs/snippets/`.
- Add redirects for moved URLs in `docs/docs.json`.
