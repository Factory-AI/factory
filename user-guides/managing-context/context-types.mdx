## Overview of Context Types

Factory supports various types of context, each serving different purposes in your development workflow. Understanding these types helps you maximize the value of context in your work.

## Project Management Context

<CardGroup cols={3}>
  <Card title="Jira & Linear Tickets" icon="ticket">

    * Comment threads and discussions

    * Status and priority tracking

    * Team and cycle information

    * Roadmap context
  </Card>

  <Card title="GitHub Issues" icon="circle-dot">
    * Issue descriptions

    * Labels and milestones

    * Discussion threads

    * Project board status

    * Related pull requests
  </Card>

  <Card title="GitLab Issues" icon="bookmark">

## Common Context Combinations

<Note>
  Effective development often requires combining multiple types of context to get a complete picture of the task at hand.
</Note>

<CardGroup cols={2}>
  <Card title="Feature Development" icon="code-branch">
    **Required Context:**

    * Project ticket (Jira/Linear)

    * Design documents (Notion/GDocs)

    * Related PRs

    * Architecture diagrams

    * Implementation guides

    **Optional Context:**

    * Team discussions

    * Related features

    * Performance requirements
  </Card>

  <Card title="Bug Resolution" icon="bug">
    **Required Context:**

    * Bug ticket

    * Error logs

    * Affected code files

    * Test cases

    * Recent changes

    **Optional Context:**

    * Similar past issues

    * Performance metrics

    * User reports
  </Card>
</CardGroup>
