Alloy (**Auto Model Routing**) automatically chooses the best model for each Droid task. Instead of manually picking a single model for every session, Alloy evaluates the task and routes work to the model that balances quality, latency, and cost.

Alloy is available in the Factory CLI and App and works with the same enterprise controls as other models.

## Why use Alloy?

Alloy is designed for teams that want strong results without manually changing model selection for every workflow.

  <Card title="Best performance at lower cost" icon="chart-line">

## Availability

Alloy is available in the Factory CLI and App model selector for select customers as **Alloy (Research Preview)**.

## Enterprise controls

Enterprise admins can manage Alloy through the same model governance system used for other Factory-supported models, including allowing or restricting Alloy at the organization level like any other model.

For more information on enterprise control of models, see [Models, LLM Gateways & Integrations](https://docs.factory.ai/enterprise/models-llm-gateways-and-integrations).

## FAQ

### How does model routing work?

Alloy is a per-task router that uses a mix of session and per-request routing. It strongly considers prompt cache maintenance and savings in its optimization.

### Can Alloy be configured?

Alloy can be enabled/disabled like any other model today. We also offer Enterprise Controls for Alloy routing guidance.

### Are there benchmarks on Alloy's performance?

Yes. Alloy maintains frontier-level performance on challenging benchmarks like Terminal Bench 2 and Legacy Bench, and provides the best balance of quality, latency, and cost.

### What are the cost benefits of Alloy?

Alloy maintains frontier-level performance while reducing costs roughly 20–25% compared with always using top-tier models.
