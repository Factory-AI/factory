Auto-Model Router (**Auto Model Routing**) automatically chooses the best model for each Droid task. Instead of manually picking a single model for every session, Auto-Model Router evaluates the task and routes work to the model that balances quality, latency, and cost.

Auto-Model Router is available in the Factory CLI and App and works with the same enterprise controls as other models.

## Why use Auto-Model Router?

Auto-Model Router is designed for teams that want strong results without manually changing model selection for every workflow.

## Availability

Auto-Model Router is available in the Factory CLI and App model selector for select customers as **Auto-Model Router (Research Preview)**.

## Enterprise controls

Enterprise admins can manage Auto-Model Router through the same model governance system used for other Factory-supported models, including allowing or restricting Auto-Model Router at the organization level like any other model.

For more information on enterprise control of models, see [Models, LLM Gateways & Integrations](https://docs.factory.ai/enterprise/models-llm-gateways-and-integrations).

## FAQ

### How does model routing work?

Auto-Model Router is a per-task router that uses a mix of session and per-request routing. It strongly considers prompt cache maintenance and savings in its optimization.

### Can Auto-Model Router be configured?

Auto-Model Router can be enabled/disabled like any other model today. We also offer Enterprise Controls for Auto-Model Router guidance.

### Are there benchmarks on Auto-Model Router's performance?

Yes. Auto-Model Router maintains frontier-level performance on challenging benchmarks like Terminal Bench 2 and Legacy Bench, and provides the best balance of quality, latency, and cost.

### What are the cost benefits of Auto-Model Router?

Auto-Model Router maintains frontier-level performance while reducing costs roughly 20–25% compared with always using top-tier models.
