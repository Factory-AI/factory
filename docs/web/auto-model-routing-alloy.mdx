Auto-Model Routing, powered by **Alloy**, automatically chooses the right model for each Droid task. Instead of manually picking a single model for every session, Alloy evaluates the task and routes work to a model that balances quality, latency, and cost.

Alloy is available in the Factory app and works with the same enterprise controls as other models, so organizations can adopt routing without giving up model governance.

## Why use Alloy?

Alloy is designed for teams that want strong results without manually tuning model selection for every workflow.

## Availability

Alloy is available from the Factory app model selector as **Auto-Model Routing (Alloy)**. When selected, Droid uses the router for supported tasks instead of pinning the entire session to one model.

Because Alloy is in Research Preview, teams should validate it on representative workflows before standardizing it across production-critical automation.

## Enterprise controls

Enterprise admins can manage Alloy through the same model governance system used for other Factory-supported models:

- Allow or restrict Alloy at the organization level.
- Control which projects and users can select it.
- Enforce existing provider, gateway, BYOK, audit, and data-handling policies.
- Keep disallowed models unavailable to the router.

Custom routing guidance for Enterprise admins is also rolling out, enabling organizations to provide policy and preference guidance that Alloy uses when selecting models.

## When to use Alloy

Use Alloy when you want Droid to optimize model selection automatically across mixed workloads, such as:

- Product implementation that alternates between planning, coding, testing, and review.
- High-volume sessions where cost efficiency matters.
- Teams that want near-frontier quality without requiring every user to understand model tradeoffs.

For one-off critical tasks where your team requires a specific model, continue selecting that model directly from the model selector.
