</Note>

Auto-Model Routing (**Alloy**) automatically chooses the best model for each Droid task. Instead of manually picking a single model for every session, Alloy evaluates the task and routes work to the model that balances quality, latency, and cost.

Alloy is available in the Factory CLI and app and works with the same enterprise controls as other models, so organizations can adopt routing without giving up model governance.

## Why use Alloy?

Alloy is designed for teams that want strong results without manually changing model selection for every workflow.

## Availability

Alloy is available in the Factory CLI and app model selector as **Alloy (Research Preview)**.

## Enterprise controls

Enterprise admins can manage Alloy through the same model governance system used for other Factory-supported models, including allowing or restricting Alloy at the organization level like any other model.

For more information on enterprise control of models, see [Models, LLM Gateways & Integrations](https://docs.factory.ai/enterprise/models-llm-gateways-and-integrations).
