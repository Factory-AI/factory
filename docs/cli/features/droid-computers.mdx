**Research Preview** — Droid Computers is currently available only to **Max plan** subscribers. There is a limit of **one computer per user** during the preview.
</Note>

Droid Computers are persistent compute environments that Droid can connect to and work on across sessions. Unlike ephemeral cloud templates that are destroyed after each session, computers **retain state** — installed packages, files, running services, and configuration all persist between sessions.

## Platform-managed computers

The fastest way to get started is to create a computer directly from the Factory web app.

1. Navigate to **Settings → Droid Computers**.
2. Click **Create**.
3. Give your computer a name.

Factory provisions a cloud machine (4 CPU, 8GB RAM) automatically. Provisioning progress is tracked live in the UI through the following steps:

1. **Creating computer** — allocating the cloud machine
2. **Setting up user** — creating the `factory-user` account with sudo access
3. **Configuring environment** — writing environment config, SSH keys, and service files
4. **Installing Droid** — downloading and installing the Droid binary
5. **Starting services** — launching the SSH and Droid daemon services

Once provisioning completes, the computer status changes to **Active** and it becomes available for use in sessions. Computers auto-pause when idle and auto-resume when a new session targets them.

## Bring Your Own Machine (BYOM)

You can register any Linux machine — cloud VM, on-prem server, WSL instance, etc. — as a Droid Computer.

### Prerequisites

- Droid CLI installed on the machine
- Authenticated with Factory (`droid login`)
- Network access to Factory APIs (no inbound ports need to be opened)

### Setup

Register the machine:

```bash
droid computer register [name]
```

The name defaults to the machine's hostname if omitted. A `computer.json` config file is written to `~/.factory/` upon registration.

Then start the daemon so Factory can connect to the machine:

```bash
droid daemon --host 0.0.0.0 --port <port> --remote-access
```

The `--remote-access` flag tells the daemon to connect through Factory's secure relay service, so no inbound ports need to be opened on your machine.

<Note>
Only one computer registration is allowed per machine at a time. Run `droid computer remove` first if you need to re-register.
</Note>

## CLI commands

| Command | Description |
|---------|-------------|
| `droid computer register [name]` | Register the current machine as a BYOM computer |
| `droid computer list` | List all computers (shows name, ID, status; marks current machine) |
| `droid computer remove` | Unregister the current machine and clean up local config |
| `droid computer ssh <name>` | Open an interactive SSH session to any computer |

### SSH options

The `ssh` subcommand supports the following flags:

- `--proxy` — Run as a stdio proxy (for use as a `ProxyCommand`)
- `--port <port>` — Target port on the remote machine (default: 22)
- `--debug` — Enable verbose connection logging

## Using computers in sessions

### Web app

In the new session modal, select the **Computer** tab, pick an active computer, optionally set a working directory (e.g., `/home/factory-user/projects/my-app`), and start your session.

Connection status (connecting / connected / error) is shown in real time with retry options.

### Slack

Computers can also be selected when creating sessions via the Slack integration.

## Monitoring

Click on any computer in **Settings → Droid Computers** to open its detail page. For platform-managed computers, the detail page shows live resource metrics:

- **CPU usage** — percentage utilization over time
- **Memory usage** — used vs. total memory
- **Disk usage** — used vs. total disk space

Metrics refresh every 10 seconds and support configurable time ranges.

The detail page also shows the current **daemon version** and connection status.

## Updating the daemon

The computer detail page displays the running daemon version alongside the latest available version. When an update is available, an **Update** button appears. Clicking it triggers a remote update — the daemon downloads the new binary, restarts, and reconnects automatically.

For BYOM machines, you can also update the Droid CLI manually by downloading the latest version and restarting the daemon process.

## SSH & IDE integration

`droid computer ssh <name>` establishes a secure WebSocket tunnel through the daemon — no direct SSH port exposure is required. Factory generates and manages a dedicated Ed25519 SSH key pair (stored in `~/.factory/.ssh/`), separate from your personal SSH keys. The public key is injected on each connection for passwordless authentication.

### VS Code Remote-SSH

You can use `droid computer ssh` as a `ProxyCommand` for VS Code Remote-SSH:

```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  StrictHostKeyChecking no
```

This lets you open a full VS Code remote session on your Droid Computer.

## Security

- **Firewall** — Platform-managed computers restrict inbound traffic to the daemon port only; all other ports are blocked by default.
- **SSH hardening** — Root login and password authentication are disabled.
- **Relay mode** — BYOM computers (and optionally platform computers) route all traffic through Factory's relay service, so no public ports are exposed.
- **API key isolation** — Each computer receives its own Factory API key for daemon authentication, which is automatically revoked when the computer is deleted.
- **Git credentials** — Configured automatically for authenticated repository access on the computer.
