## インストール

<CodeGroup>```bash macOS/Linux
curl -fsSL https://app.factory.ai/cli | sh
```

```powershell Windows
irm https://app.factory.ai/cli/windows | iex
```</CodeGroup>

CLIは2つのモードで動作します：

- **インタラクティブ（`droid`）** - スラッシュコマンド付きのチャット優先REPL
- **非インタラクティブ（`droid exec`）** - 自動化とスクリプト用の単発実行

## CLIコマンド

| コマンド                          | 説明                                 | 例                                         |
| :------------------------------- | :------------------------------------------ | :---------------------------------------------- |
| `droid`                          | インタラクティブREPLを開始                      | `droid`                                         |
| `droid "query"`                  | 初期プロンプト付きでREPLを開始              | `droid "このプロジェクトを説明して"`                  |
| `droid exec "query"`             | インタラクティブモードなしでタスクを実行       | `droid exec "src/authを要約して"`               |
| `droid exec -f prompt.md`        | ファイルからプロンプトを読み込み                       | `droid exec -f .factory/prompts/review.md`      |
| `cat file \| droid exec`         | パイプされたコンテンツを処理                       | `git diff \| droid exec "リリースノートを下書き"` |
| `droid exec -s <id> "query"`     | execモードで既存セッションを再開        | `droid exec -s session-123 "続けて"`          |
| `droid exec --list-tools`        | 利用可能なツールをリストして終了             | `droid exec --list-tools`                       |
| `droid update`                   | CLIを手動で最新バージョンに更新   | `droid update`                                  |

## CLIフラグ

コマンドラインフラグでdroidの動作をカスタマイズ：

| フラグ                                 | 説明                                                                                                  | 例                                                    |
| :----------------------------------- | :----------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------- |
| `-f, --file <path>`                  | ファイルからプロンプトを読み込み                                                                                      | `droid exec -f plan.md`                                    |
| `-m, --model <id>`                   | 特定のモデルを選択（[モデルID](#available-models)を参照）                                                 | `droid exec -m claude-opus-4-6`                            |
| `-s, --session-id <id>`              | 既存セッションを継続                                                                                 | `droid exec -s session-abc123`                             |
| `--auto <level>`                     | [自律レベル](#autonomy-levels)を設定（`low`、`medium`、`high`）                                             | `droid exec --auto medium "テストを実行"`                     |
| `--enabled-tools <ids>`              | 特定のツールを強制有効化（カンマまたはスペース区切り）                                                       | `droid exec --enabled-tools ApplyPatch,Bash`               |
| `--disabled-tools <ids>`             | この実行で特定のツールを無効化                                                                          | `droid exec --disabled-tools execute-cli`                  |
| `--list-tools`                       | 利用可能なツールを表示して終了                                                                               | `droid exec --list-tools`                                  |
| `-o, --output-format <format>`       | 出力フォーマット（`text`、`json`、`stream-json`、`stream-jsonrpc`）                                              | `droid exec -o json "APIを文書化"`                        |
| `--input-format <format>`            | 入力フォーマット（マルチターン用の`stream-json`、`stream-jsonrpc`）                                                | `droid exec --input-format stream-jsonrpc -o stream-jsonrpc` |
| `-r, --reasoning-effort <level>`     | 推論労力をオーバーライド（`off`、`none`、`low`、`medium`、`high`）                                           | `droid exec -r high "不安定なテストをデバッグ"`                    |
| `--spec-model <id>`                  | 仕様計画に異なるモデルを使用                                                             | `droid exec --spec-model claude-sonnet-4-5-20250929`       |
| `--spec-reasoning-effort <level>`    | specモードの推論労力をオーバーライド                                                                      | `droid exec --use-spec --spec-reasoning-effort high`       |
| `--use-spec`                         | 仕様モードで開始（実行前に計画）                                                          | `droid exec --use-spec "ユーザープロファイルを追加"`                |
| `--skip-permissions-unsafe`          | すべての権限プロンプトをスキップ（⚠️ 細心の注意を払って使用）                                                    | `droid exec --skip-permissions-unsafe`                     |
| `--cwd <path>`                       | 特定の作業ディレクトリから実行                                                                    | `droid exec --cwd ../service "テストを実行"`                  |
| `--tag <spec>`                       | セッションタグ（名前またはJSON、繰り返し可能）                                                                       | `droid exec --tag code-review`                             |
| `--log-group-id <id>`                | ログフィルタリング用のロググループID                                                                              | `droid exec --log-group-id grp-123`                        |
| `-v, --version`                      | CLIバージョンを表示                                                                                          | `droid -v`                                                 |
| `-h, --help`                         | ヘルプ情報を表示                                                                                        | `droid --help`                                             |

<Tip>
  スクリプトと自動化には`--output-format json`を使用し、droidの応答をプログラム的に解析できます。
</Tip>

## 自律レベル

`droid exec`は階層化された自律性を使用して、エージェントが実行できる操作を制御します。環境が安全な場合にのみアクセスレベルを上げてください。

| レベル                         | 対象             | 主要な許可事項                                                                 |
| :---------------------------- | :----------------------- | :--------------------------------------------------------------------------------- |
| *(デフォルト)*                   | 読み取り専用偵察 | ファイル読み取り、git diff、環境検査                                      |
| `--auto low`                  | 安全な編集               | ファイル作成/編集、フォーマッタ実行、非破壊的コマンド                        |
| `--auto medium`               | ローカル開発        | 依存関係インストール、ビルド/テスト、ローカルgitコミット                                |
| `--auto high`                 | CI/CDとオーケストレーション    | git push、デプロイスクリプト、長時間実行操作                                  |
| `--skip-permissions-unsafe`   | 分離されたサンドボックスのみ  | すべてのガードレールを削除（⚠️ 使い捨てコンテナでのみ使用）                      |

**例：**```bash
# Default (read-only)
droid exec "Analyze the auth system and create a plan"

# Low autonomy - safe edits
droid exec --auto low "Add JSDoc comments to all functions"

# Medium autonomy - development work
droid exec --auto medium "Install deps, run tests, fix issues"

# High autonomy - deployment
droid exec --auto high "Run tests, commit, and push changes"
```<Warning>
  `--skip-permissions-unsafe`はすべての安全チェックを削除します。Dockerコンテナなどの分離された環境で**のみ**使用してください。
</Warning>

## インタラクティブモード機能

### Bashモード

入力が空の状態で`!`を押すとBashモードが切り替わります。BashモードではAIの解釈を通さずにコマンドが直接シェルで実行されるため、`git status`の確認や`npm test`の実行などの簡単な操作に便利です。

- **オン切り替え:** `!`を押す（入力が空の状態で）
- **コマンド実行:** シェルコマンドを入力してEnterを押す
- **オフ切り替え:** `Esc`を押して通常のAIチャットモードに戻る

Bashモードが有効になると、プロンプトが`>`から`$`に変わります。

### MCPコマンドリファレンス

`/mcp`スラッシュコマンドは、MCPサーバーの閲覧と管理のためのインタラクティブマネージャーUIを開きます。

**クイックスタート：** `/mcp`と入力し、**「Add from Registry」**を選択して40以上の事前設定済みサーバー（Linear、Sentry、Notion、Stripe、Vercel等）を閲覧します。サーバーを選択し、必要に応じて認証を行えば準備完了です。

**CLIコマンド**（スクリプト作成と自動化用）：```bash
droid mcp add <name> <url> --type http    # Add HTTP server
droid mcp add <name> "<command>"          # Add stdio server
droid mcp remove <name>                   # Remove a server
```[MCP設定](/jp/cli/configuration/mcp)で、完全なレジストリリスト、CLIオプション（`--env`、`--header`）、設定ファイル、ユーザーとプロジェクト設定のレイヤー化の仕組みについて確認してください。

## 認証

1. [app.factory.ai/settings/api-keys](https://app.factory.ai/settings/api-keys)でAPIキーを生成します
2. 環境変数を設定します：

<CodeGroup>```bash macOS/Linux
export FACTORY_API_KEY=fk-...
```

```powershell Windows (PowerShell)
$env:FACTORY_API_KEY="fk-..."
```

```cmd Windows (CMD)
set FACTORY_API_KEY=fk-...
```</CodeGroup>

**変数を永続化**するには、シェルプロファイル（`~/.bashrc`、`~/.zshrc`、またはPowerShellの`$PROFILE`）に追加して長期利用できるようにします。

<Warning>
  API キーをソースコントロールにコミットしないでください。環境変数または安全なシークレット管理を使用してください。
</Warning>

## 一般的なワークフロー

### コードレビュー```bash
# Interactive review workflow
> /review

# Analysis via exec (non-interactive)
droid exec "Review this PR for security issues"

# With modifications
droid exec --auto low "Review code and add missing type hints"
```[コードレビュー文書](/jp/cli/features/code-review)で、レビュータイプ、ワークフロー、ベストプラクティスの詳細なガイダンスをご覧ください。

### テストとデバッグ```bash
# Investigation
droid exec "Analyze failing tests and explain root cause"

# Fix and verify
droid exec --auto medium "Fix failing tests and run test suite"
```### リファクタリング```bash
# Planning
droid exec "Create refactoring plan for auth module"

# Execution
droid exec --auto low --use-spec "Refactor auth module"
```### CI/CD統合```yaml
# GitHub Actions example
- name: Run Droid Analysis
  env:
    FACTORY_API_KEY: ${{ secrets.FACTORY_API_KEY }}
  run: |
    droid exec --auto medium -f .github/prompts/deploy.md
