## インストール

<CodeGroup>```bash macOS/Linux
curl -fsSL https://app.factory.ai/cli | sh
```

```powershell Windows
irm https://app.factory.ai/cli/windows | iex
```</CodeGroup>

CLIは2つのモードで動作します：

- **インタラクティブ (`droid`)** - スラッシュコマンド付きのチャット優先REPL
- **非インタラクティブ (`droid exec`)** - 自動化とスクリプト用のワンショット実行

## CLIコマンド

| コマンド                          | 説明                                 | 例                                         |
| :------------------------------- | :---------------------------------- | :---------------------------------------------- |
| `droid`                          | インタラクティブREPLを開始                      | `droid`                                         |
| `droid "query"`                  | 初期プロンプト付きでREPLを開始              | `droid "explain this project"`                  |
| `droid exec "query"`             | インタラクティブモードなしでタスクを実行       | `droid exec "summarize src/auth"`               |
| `droid exec -f prompt.md`        | ファイルからプロンプトを読み込み                       | `droid exec -f .factory/prompts/review.md`      |
| `cat file \| droid exec`         | パイプされたコンテンツを処理                       | `git diff \| droid exec "draft release notes"` |
| `droid exec -s <id> "query"`     | execモードで既存セッションを再開        | `droid exec -s session-123 "continue"`          |
| `droid exec --list-tools`        | 利用可能なツールをリストして終了             | `droid exec --list-tools`                       |
| `droid update`                   | CLIを最新バージョンに手動更新   | `droid update`                                  |

## CLIフラグ

コマンドラインフラグでdroidの動作をカスタマイズできます：

| フラグ                                 | 説明                                                                                                  | 例                                                    |
| :----------------------------------- | :----------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------- |
| `-f, --file <path>`                  | ファイルからプロンプトを読み取り                                                                                      | `droid exec -f plan.md`                                    |
| `-m, --model <id>`                   | 特定のモデルを選択（[モデルID](#available-models)を参照）                                                 | `droid exec -m claude-opus-4-6`                            |
| `-s, --session-id <id>`              | 既存セッションを継続                                                                                 | `droid exec -s session-abc123`                             |
| `--auto <level>`                     | [自律性レベル](#autonomy-levels)を設定（`low`、`medium`、`high`）                                             | `droid exec --auto medium "run tests"`                     |
| `--enabled-tools <ids>`              | 特定のツールを強制有効化（カンマまたはスペース区切り）                                                       | `droid exec --enabled-tools ApplyPatch,Bash`               |
| `--disabled-tools <ids>`             | この実行で特定のツールを無効化                                                                          | `droid exec --disabled-tools execute-cli`                  |
| `--list-tools`                       | 利用可能なツールを表示して終了                                                                               | `droid exec --list-tools`                                  |
| `-o, --output-format <format>`       | 出力形式（`text`、`json`、`stream-json`、`stream-jsonrpc`）                                              | `droid exec -o json "document API"`                        |
| `--input-format <format>`            | 入力形式（マルチターン用の`stream-json`、`stream-jsonrpc`）                                                | `droid exec --input-format stream-jsonrpc -o stream-jsonrpc` |
| `-r, --reasoning-effort <level>`     | 推論努力レベルをオーバーライド（`off`、`none`、`low`、`medium`、`high`）                                           | `droid exec -r high "debug flaky test"`                    |
| `--spec-model <id>`                  | 仕様計画に異なるモデルを使用                                                             | `droid exec --spec-model claude-sonnet-4-5-20250929`       |
| `--spec-reasoning-effort <level>`    | 仕様モードの推論努力レベルをオーバーライド                                                                      | `droid exec --use-spec --spec-reasoning-effort high`       |
| `--use-spec`                         | 仕様モードで開始（実行前に計画）                                                          | `droid exec --use-spec "add user profiles"`                |
| `--skip-permissions-unsafe`          | すべての権限プロンプトをスキップ（⚠️ 極度の注意で使用）                                                    | `droid exec --skip-permissions-unsafe`                     |
| `--cwd <path>`                       | 特定の作業ディレクトリから実行                                                                    | `droid exec --cwd ../service "run tests"`                  |
| `--tag <spec>`                       | セッションタグ（名前またはJSON、反復可能）                                                                       | `droid exec --tag code-review`                             |
| `--log-group-id <id>`                | ログフィルタリング用のロググループID                                                                              | `droid exec --log-group-id grp-123`                        |
| `-v, --version`                      | CLIバージョンを表示                                                                                          | `droid -v`                                                 |
| `-h, --help`                         | ヘルプ情報を表示                                                                                        | `droid --help`                                             |

<Tip>
  スクリプトと自動化には`--output-format json`を使用して、droidの応答をプログラム的に解析できます。
</Tip>

## 自律性レベル

`droid exec`は階層化された自律性を使用して、エージェントが実行できる操作を制御します。環境が安全な場合にのみアクセスレベルを上げてください。

| レベル                         | 想定用途             | 主な許可内容                                                                 |
| :---------------------------- | :----------------------- | :--------------------------------------------------------------------------------- |
| *（デフォルト）*                   | 読み取り専用偵察 | ファイル読み取り、git diff、環境検査                                      |
| `--auto low`                  | 安全な編集               | ファイル作成・編集、フォーマッター実行、非破壊的コマンド                        |
| `--auto medium`               | ローカル開発        | 依存関係インストール、ビルド・テスト、ローカルgitコミット                                |
| `--auto high`                 | CI/CD・オーケストレーション    | Git push、デプロイスクリプト、長時間実行操作                                  |
| `--skip-permissions-unsafe`   | 分離サンドボックスのみ  | すべてのガードレールを削除（⚠️ 使い捨てコンテナでのみ使用）                      |

**例：**```bash
# Default (read-only)
droid exec "Analyze the auth system and create a plan"

# Low autonomy - safe edits
droid exec --auto low "Add JSDoc comments to all functions"

# Medium autonomy - development work
droid exec --auto medium "Install deps, run tests, fix issues"

# High autonomy - deployment
droid exec --auto high "Run tests, commit, and push changes"
```<Warning>
  `--skip-permissions-unsafe` は全ての安全性チェックを削除します。Dockerコンテナのような隔離された環境で**のみ**使用してください。
</Warning>

## インタラクティブモードの機能

### Bashモード

入力が空の時に `!` を押してbashモードを切り替えます。bashモードでは、コマンドがAIの解釈なしに直接シェルで実行されます—`git status` の確認や `npm test` の実行などの簡単な操作に便利です。

- **オンに切り替え:** `!` を押す（入力が空の時）
- **コマンドを実行:** 任意のシェルコマンドを入力してEnterを押す
- **オフに切り替え:** `Esc` を押して通常のAIチャットモードに戻る

bashモードがアクティブな時、プロンプトが `>` から `$` に変わります。

### MCPコマンドリファレンス

`/mcp` スラッシュコマンドは、MCPサーバーの閲覧と管理用のインタラクティブマネージャーUIを開きます。

**クイックスタート:** `/mcp` と入力し、**"Add from Registry"** を選択して40以上の事前設定されたサーバー（Linear、Sentry、Notion、Stripe、Vercel、その他）を閲覧します。サーバーを選択し、必要に応じて認証を行えば、使用準備完了です。

**CLI コマンド** スクリプトと自動化用：```bash
droid mcp add <name> <url> --type http    # Add HTTP server
droid mcp add <name> "<command>"          # Add stdio server
droid mcp remove <name>                   # Remove a server
```[MCP設定](/jp/cli/configuration/mcp)で、完全なレジストリリスト、CLIオプション（`--env`、`--header`）、設定ファイル、およびユーザー設定とプロジェクト設定のレイヤー構造について確認してください。

## 認証

1. [app.factory.ai/settings/api-keys](https://app.factory.ai/settings/api-keys)でAPIキーを生成
2. 環境変数を設定：

<CodeGroup>```bash macOS/Linux
export FACTORY_API_KEY=fk-...
```

```powershell Windows (PowerShell)
$env:FACTORY_API_KEY="fk-..."
```

```cmd Windows (CMD)
set FACTORY_API_KEY=fk-...
```</CodeGroup>

**変数を永続化**するには、シェルプロファイル（`~/.bashrc`、`~/.zshrc`、またはPowerShell `$PROFILE`）で長期使用向けに設定してください。

<Warning>
  APIキーをソースコントロールにコミットしないでください。環境変数またはセキュアなシークレット管理を使用してください。
</Warning>

## 一般的なワークフロー

### コードレビュー```bash
# Interactive review workflow
> /review

# Analysis via exec (non-interactive)
droid exec "Review this PR for security issues"

# With modifications
droid exec --auto low "Review code and add missing type hints"
```[コードレビューのドキュメント](/jp/cli/features/code-review)で、レビューの種類、ワークフロー、ベストプラクティスについての詳細なガイダンスを参照してください。

### テストとデバッグ```bash
# Investigation
droid exec "Analyze failing tests and explain root cause"

# Fix and verify
droid exec --auto medium "Fix failing tests and run test suite"
```### リファクタリング```bash
# Planning
droid exec "Create refactoring plan for auth module"

# Execution
droid exec --auto low --use-spec "Refactor auth module"
```### CI/CDインテグレーション```yaml
# GitHub Actions example
- name: Run Droid Analysis
  env:
    FACTORY_API_KEY: ${{ secrets.FACTORY_API_KEY }}
  run: |
    droid exec --auto medium -f .github/prompts/deploy.md
