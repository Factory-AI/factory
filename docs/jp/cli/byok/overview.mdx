Factory CLI は BYOK（Bring Your Own Key）を通じてカスタムモデル設定をサポートしています。独自の OpenAI や Anthropic のキーを使用したり、任意のオープンソースモデルプロバイダーに接続したり、ハードウェア上でローカルにモデルを実行したりできます。設定が完了したら、`/model` コマンドを使用してモデル間を切り替えることができます。

<Note>
  API キーはローカルに保持され、Factory サーバーにはアップロードされません。カスタムモデルは CLI でのみ利用可能で、Factory のウェブやモバイルプラットフォームには表示されません。
</Note>

## 設定リファレンス

`~/.factory/settings.json` の `customModels` 配列にカスタムモデルを追加します：```json
{
  "customModels": [
    {
      "model": "your-model-id",
      "displayName": "My Custom Model",
      "baseUrl": "https://api.provider.com/v1",
      "apiKey": "${PROVIDER_API_KEY}",
      "provider": "generic-chat-completion-api",
      "maxOutputTokens": 16384
    }
  ]
}
```<Tip>
  `settings.json`（および`settings.local.json`）では、`apiKey`は`${VAR_NAME}`構文を使用した環境変数参照をサポートしています。例えば、`"apiKey": "${PROVIDER_API_KEY}"`は`PROVIDER_API_KEY`という名前の環境変数から読み取ります（例：`export PROVIDER_API_KEY=your_key_here`）。
</Tip>

<Note>
  **レガシーサポート**: snake_caseフィールド名（`custom_models`、`base_url`など）を使用した`~/.factory/config.json`のカスタムモデルは、下位互換性のために引き続きサポートされています。両方のファイルが読み込まれマージされ、`settings.json`が優先されます。`apiKey`の環境変数展開は`settings.json`/`settings.local.json`に適用され、レガシーの`config.json`には適用されません。
</Note>

### サポートされるフィールド

| フィールド | 型 | 必須 | 説明 |
|-------|------|----------|-------------|
| `model` | `string` | ✓ | API経由で送信されるモデル識別子（例：`claude-sonnet-4-5-20250929`、`gpt-5-codex`、`qwen3:4b`） |
| `displayName` | `string` | | モデル選択画面に表示される人間にとって分かりやすい名前 |
| `baseUrl` | `string` | ✓ | APIエンドポイントのベースURL |
| `apiKey` | `string` | ✓ | プロバイダー用のAPIキー。空にはできません。`settings.json`/`settings.local.json`で`${VAR_NAME}`をサポートします（例：`${PROVIDER_API_KEY}`は`PROVIDER_API_KEY`環境変数を使用します）。 |
| `provider` | `string` | ✓ | 次のいずれか：`anthropic`、`openai`、または`generic-chat-completion-api` |
| `maxOutputTokens` | `number` | | モデルレスポンスの最大出力トークン数 |
| `noImageSupport` | `boolean` | | このモデルの画像入力を無効にするには`true`に設定 |
| `extraArgs` | `object` | | APIリクエストに含める追加のプロバイダー固有の引数 |
| `extraHeaders` | `object` | | リクエストと一緒に送信する追加のHTTPヘッダー |

### extraArgsの使用

temperatureやtop_pなどのプロバイダー固有のパラメーターを渡します：```json
{
  "customModels": [
    {
      "model": "your-model",
      "displayName": "Custom Model",
      "baseUrl": "https://your-provider.com/v1",
      "apiKey": "YOUR_API_KEY",
      "provider": "generic-chat-completion-api",
      "extraArgs": {
        "temperature": 0.7,
        "top_p": 0.9
      }
    }
  ]
}
```### extraHeaders の使用

API リクエストにカスタム HTTP ヘッダーを追加する：```json
{
  "customModels": [
    {
      "model": "your-model",
      "displayName": "Custom Model",
      "baseUrl": "https://your-provider.com/v1",
      "apiKey": "YOUR_API_KEY",
      "provider": "generic-chat-completion-api",
      "extraHeaders": {
        "X-Custom-Header": "value",
        "Authorization": "Bearer YOUR_TOKEN"
      }
    }
  ]
}
```---

## プロバイダーの理解

Factoryは、API互換性を決定する3つのプロバイダータイプをサポートしています：

| プロバイダー | APIフォーマット | 用途 | ドキュメント |
|----------|------------|---------|---------------|
| `anthropic` | Anthropic Messages API (v1/messages) | 公式APIまたは互換プロキシ上のAnthropicモデル | [Anthropic Messages API](https://docs.claude.com/en/api/messages) |
| `openai` | OpenAI Responses API | 公式APIまたは互換プロキシ上のOpenAIモデル。GPT-5やGPT-5-Codexなどの最新モデルに必要。 | [OpenAI Responses API](https://platform.openai.com/docs/api-reference/responses) |
| `generic-chat-completion-api` | OpenAI Chat Completions API | OpenRouter、Fireworks、Together AI、Ollama、vLLM、およびほとんどのオープンソースプロバイダー | [OpenAI Chat Completions API](https://platform.openai.com/docs/api-reference/chat) |

<Warning>
  Factoryは人気モデルでのDroidのパフォーマンスを積極的に検証していますが、すべてのカスタムモデルが開箱即用で動作することは保証できません。公式APIを通じてアクセスされるAnthropicおよびOpenAIモデルのみが完全にテストされ、ベンチマークされています。
</Warning>

<Note>
  **モデルサイズの考慮**: 300億パラメータ以下のモデルは、エージェンティックなコーディングタスクにおいて著しく低いパフォーマンスを示しています。これらの小さなモデルは実験や学習には有用ですが、本番環境でのコーディング作業や複雑なソフトウェアエンジニアリングタスクには一般的に推奨されません。
</Note>

## プロンプトキャッシュ

Factory CLIは、利用可能な場合にプロンプトキャッシュを自動的に使用してAPIコストを削減します：

- **公式プロバイダー（`anthropic`、`openai`）**: Factoryは公式APIを通じてプロンプトキャッシュを使用しようとします。キャッシュ動作は各プロバイダーの実装と要件に従います。
- **汎用プロバイダー（`generic-chat-completion-api`）**: プロンプトキャッシュサポートはプロバイダーによって異なり、保証できません。一部のプロバイダーはキャッシュをサポートする場合がありますが、そうでない場合もあります。

### プロンプトキャッシュの確認

カスタムモデルでプロンプトキャッシュが正しく動作しているかを確認するには：

1. カスタムモデルで会話を実行する
2. Droid CLIで `/cost` コマンドを使用してコストの内訳を表示する
3. 出力でキャッシュヒット率と節約を確認する

期待されるキャッシュ節約が見られない場合は、プロバイダーのプロンプトキャッシュサポートと要件に関するドキュメントを参照してください。

## カスタムモデルの使用

設定後、CLIでカスタムモデルにアクセス：

1. `/model` コマンドを使用する
2. カスタムモデルはFactory提供モデルの下の別の「カスタムモデル」セクションに表示される
3. 任意のモデルを選択して使用開始

カスタムモデルは `displayName` で設定した名前で表示されるため、異なるプロバイダーと設定を簡単に識別できます。

## トラブルシューティング

### モデルがセレクターに表示されない
- `~/.factory/settings.json`（レガシーフォーマットを使用している場合は `config.json`）のJSON構文を確認
- 設定変更はファイル監視を通じて自動的に検出される
- すべての必須フィールドが存在することを確認

### 「無効なプロバイダー」エラー
- プロバイダーは正確に `anthropic`、`openai`、または `generic-chat-completion-api` である必要がある
- タイプミスを確認し、適切な大文字小文字を使用していることを確認

### 認証エラー
- APIキーが有効で利用可能なクレジットがあることを確認
- APIキーに適切な権限があることを確認
- ベースURLがプロバイダーのドキュメントと一致することを確認

### ローカルモデルが接続できない
- ローカルサーバーが実行中であることを確認（例：`ollama serve`）
- ベースURLが正しく、必要に応じて `/v1/` サフィックスが含まれていることを確認
- モデルがローカルでプル/利用可能であることを確認

### レート制限またはクォータエラー
- プロバイダーのレート制限と使用クォータを確認
- プロバイダーのダッシュボードを通じて使用状況を監視
