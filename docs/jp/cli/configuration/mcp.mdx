Model Context Protocol (MCP) サーバーは、追加のツールとコンテキストを提供することで droid の機能を拡張します。Droid はそれらを管理する 2 つの方法を提供します：簡単なブラウジングとセットアップのためのインタラクティブ UI、またはスクリプトと自動化のための CLI コマンドです。

## クイックスタート: レジストリから追加

始めるための最も簡単な方法は、内蔵のレジストリを使用することです。droid で `/mcp` と入力し、**「レジストリから追加」** を選択して、40以上の事前設定されたサーバーをブラウズできます：

| サーバー | 説明 |
| :----- | :---------- |
| figma | デザインの探索と実装 |
| linear | 課題追跡とプロジェクト管理 |
| sentry | エラー追跡とパフォーマンス監視 |
| notion | ノート、ドキュメント、プロジェクト管理 |
| supabase | Supabase プロジェクトの作成と管理 |
| stripe | 決済処理 API |
| vercel | プロジェクトとデプロイメントの管理 |
| playwright | エンドツーエンドブラウザテスト |
| hubspot | CRM データ管理 |
| mongodb | データベース管理 |
| ... | その他多数 |

リストからサーバーを選択し、必要に応じて認証を行い（ほとんどの HTTP サーバーは OAuth をサポート—ブラウザのプロンプトに従うだけです）、サーバーが使用可能になります。

<Tip>
  レジストリは始めるための最も速い方法です。カスタムサーバーや自動化には、以下の CLI コマンドを使用してください。
</Tip>

## インタラクティブマネージャー (`/mcp`)

droid 内で `/mcp` と入力すると、インタラクティブな MCP マネージャーが開きます。ここから以下のことができます：

- **サーバーをブラウズ** - すべての設定済みサーバーとその接続状態を確認
- **ツールを表示** - 接続された各サーバーが提供するツールを調査
- **有効/無効化** - サーバーを削除せずに一時的に無効化
- **認証** - ブラウザ経由で OAuth 対応サーバーに接続
- **認証をクリア** - サーバーの保存された認証情報を削除
- **レジストリから追加** - 人気の MCP サーバーのワンクリック設定
- **サーバーを削除** - ユーザー設定のサーバーを削除

## CLI 経由でサーバーを追加

スクリプトと自動化には、`droid mcp add` を使用します。Droid は 2 種類のサーバーをサポートします：**http**（リモートエンドポイント）と **stdio**（ローカルプロセス）。

### HTTP サーバーの追加

HTTP サーバーはリモートの MCP エンドポイントです - クラウドサービスと API に接続する推奨方法です。

**構文:**```bash
droid mcp add <name> <url> --type http [--header "KEY: VALUE"...]
```**引数:**

- `name` - 一意のサーバー識別子
- `url` - MCPサーバーのHTTP/HTTPS URL
- `--type http` - HTTPトランスポートを指定するために必要なフラグ
- `--header "KEY: VALUE"` - 認証用のHTTPヘッダー（複数回使用可能）

### Stdioサーバーの追加

Stdioサーバーはあなたのマシン上でローカルプロセスとして実行されます - 直接的なシステムアクセスが必要なツールに最適です。

**構文:**```bash
droid mcp add <name> "<command>" [--env KEY=VALUE...]
```**引数:**

- `name` - 一意のサーバー識別子
- `command` - サーバーを起動するコマンド（スペースが含まれる場合は引用符で囲む）
- `--env KEY=VALUE` - 環境変数（複数回使用可能）

## 設定

MCPサーバーの設定は2つのレベルで保存できます：

| レベル | 場所 | 目的 |
| :---- | :------- | :------ |
| **ユーザー** | `~/.factory/mcp.json` | 個人用サーバー、すべてのプロジェクトで利用可能 |
| **プロジェクト** | `.factory/mcp.json` | チーム共有サーバー、リポジトリにコミット |

### レイヤリングの仕組み

ユーザー設定がプロジェクト設定よりも優先されます。両方のレベルで同じサーバーが定義されている場合、ユーザーレベルの設定が優先されます。

**主要な動作：**
- プロジェクトで定義されたサーバーを**有効化・無効化**すると、新しい状態でユーザー設定にコピーが保存されます。元のプロジェクト設定は変更されないため、チームメイトには影響しません。
- **プロジェクトサーバーはCLIや`/mcp` UIから削除できません**。削除するには、`.factory/mcp.json`を直接編集してください。
- `droid mcp add`やレジストリから追加したサーバーは、常に**ユーザー設定**に保存されます。

### OAuthトークン

OAuthトークンは、プロジェクトごとではなく、システムキーリング（またはフォールバックファイル）にグローバルに保存されます。あるプロジェクトでサーバーの認証を行うと、そのサーバーが設定されているすべての場所で認証済みになります。

サーバーの認証をクリアするには、`/mcp`インタラクティブマネージャーを使用して「Clear Auth」を選択してください。

### 設定スキーマ

各サーバーエントリには以下が含まれます：

| フィールド | タイプ | 説明 |
| :---- | :--- | :---------- |
| `type` | `"stdio" \| "http"` | サーバータイプ |
| `disabled` | `boolean` | サーバーを一時的に無効にする（デフォルト：`false`） |
| `disabledTools` | `string[]` | このMCPサーバーで無効にするツール名のリスト |

**stdio**サーバー用：
- **command**: 実行する実行可能ファイル
- **args**: コマンドライン引数（配列）
- **env**: 環境変数（オブジェクト）

**http**サーバー用：
- **url**: HTTP/HTTPSエンドポイントURL
- **headers**: 認証用HTTPヘッダー（オブジェクト）

**`mcp.json`の例：**```json
{
  "mcpServers": {
    "linear": {
      "type": "http",
      "url": "https://mcp.linear.app/mcp",
      "disabled": false
    },
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"],
      "disabled": false
    }
  }
}
```Droidは設定ファイルが変更されると自動的にリロードするため、サーバーを追加した後すぐに利用可能になります。
