## 設定へのアクセス

droidの設定を行うには：

1. `droid` を実行
2. `/settings` を入力
3. 対話形式で設定を調整

変更は即座に反映され、設定ファイルに保存されます。

## 設定の保存場所

| OS        | 場所                               |
| --------- | -------------------------------------- |
| macOS / Linux | `~/.factory/settings.json` |
| Windows   | `%USERPROFILE%\.factory\settings.json` |

ファイルが存在しない場合、初回の**droid**実行時にデフォルト値で作成されます。

### ローカルオーバーライド

任意の `.factory/` フォルダ内で `settings.local.json` を `settings.json` と並べて作成できます：

- `~/.factory/settings.local.json`（ユーザーレベル）
- `<project>/.factory/settings.local.json`（プロジェクトレベル）

ローカルオーバーライドは同じレベルの対応する `settings.json` に上書きでマージされ、同じ階層の優先順位に従います。マシン固有の設定をバージョン管理から除外したい場合は、`settings.local.json` を `.gitignore` に追加してください。

## 利用可能な設定

| 設定 | オプション | デフォルト | 説明 |
| ------- | ------- | ------- | ----------- |
| `model` | `opus`, `opus-4-6`, `opus-4-6-fast`, `sonnet`, `sonnet-4-6`, `gpt-5.4`, `gpt-5.2`, `gpt-5.2-codex`, `gpt-5.3-codex`, `haiku`, `gemini-3.1-pro`, `gemini-3-flash`, `droid-core`, `glm-5`, `kimi-k2.5`, `minimax-m2.5`, `custom-model` | `opus` | droidで使用されるデフォルトのAIモデル |
| `reasoningEffort` | `off`, `none`, `low`, `medium`, `high`（モデルによって利用可能性が異なる） | モデル依存のデフォルト | モデルが実行する構造化思考の量を制御します。 |
| `autonomyMode` | `normal`, `spec`, `auto-low`, `auto-medium`, `auto-high` | `normal` | droid開始時のデフォルト自律モードを設定します。 |
| `cloudSessionSync` | `true`, `false` | `true` | CLIセッションをFactory webにミラー。 |
| `diffMode` | `github`, `unified` | `github` | 分割されたGitHubスタイルのdiffと単一列表示の選択。 |
| `completionSound` | `off`, `bell`, `fx-ok01`, `fx-ack01`、またはカスタムファイルパス | `fx-ok01` | 応答完了時の音声キュー。 |
| `awaitingInputSound` | `off`, `bell`, `fx-ok01`, `fx-ack01`、またはカスタムファイルパス | `fx-ack01` | droidがユーザー入力を待機中の音声キュー。 |
| `soundFocusMode` | `always`, `focused`, `unfocused` | `always` | 音声通知を再生するタイミング。 |
| `commandAllowlist` | コマンドの配列 | 安全なデフォルトを提供 | 追加確認なしで実行されるコマンド。 |
| `commandDenylist` | コマンドの配列 | 制限的なデフォルトを提供 | 常に確認が必要なコマンド。 |
| `includeCoAuthoredByDroid` | `true`, `false` | `true` | コミットにDroidの共著者トレーラーを自動追加。 |
| `enableDroidShield` | `true`, `false` | `true` | シークレットスキャンとgitガードレールを有効化。 |
| `hooksDisabled` | `true`, `false` | `false` | すべてのフック実行をグローバルで無効化。 |
| `ideAutoConnect` | `true`, `false` | `false` | 外部ターミナルからIDEに自動接続。 |
| `todoDisplayMode` | `inline`, `pinned` | `pinned` | UIでのTodoリストの表示方法。 |
| `showThinkingInMainView` | `true`, `false` | `false` | メインチャット画面でAIの思考/推論ブロックを表示。 |
| `customModels` | モデル設定の配列 | `[]` | BYOKのカスタムモデル設定。[BYOK文書](/jp/cli/configuration/byok)を参照。 |

### モデル

droidを動かすデフォルトのAIモデルを選択：

- **`opus`** - Claude Opus 4.5（現在のデフォルト）
- **`opus-4-6`** - Claude Opus 4.6、最新のフラッグシップでMax推論機能付き
- **`opus-4-6-fast`** - Claude Opus 4.6 Fast、高速応答用に調整
- **`sonnet`** - Claude Sonnet 4.5、コストと品質のバランス
- **`sonnet-4-6`** - Claude Sonnet 4.6、Sonnet価格でMax推論機能
- **`gpt-5.4`** - GPT-5.4、922Kコンテキストと Extra High推論機能を持つ最新OpenAIモデル
- **`gpt-5.2`** - OpenAI GPT-5.2
- **`gpt-5.2-codex`** - GPT-5.2-Codex、Extra High推論機能付きOpenAIコーディングモデル
- **`gpt-5.3-codex`** - GPT-5.3-Codex、Extra High推論とverbosityサポート付きの最新OpenAIコーディングモデル
- **`haiku`** - Claude Haiku 4.5、高速で費用対効果の高いモデル
- **`gemini-3.1-pro`** - Gemini 3.1 Pro
- **`gemini-3-flash`** - Gemini 3 Flash、高速で安価（0.2倍の乗数）
- **`droid-core`** - GLM-4.7 オープンソースモデル
- **`glm-5`** - GLM-5 オープンソースモデル
- **`kimi-k2.5`** - Kimi K2.5 画像サポート付きオープンソースモデル
- **`minimax-m2.5`** - MiniMax M2.5 推論サポート付きオープンソースモデル（0.12倍の乗数）
- **`custom-model`** - BYOKで設定したカスタムモデル

[カスタムモデルとBYOKも追加できます。](/jp/cli/configuration/byok)

### 推論の努力

`reasoningEffort` は応答前にモデルが実行する構造化思考の量を調整します。利用可能な値はモデルに依存しますが、一般的には以下が含まれます：

- **`off` / `none`** – 構造化推論を無効化（最高速）。
- **`low`**, **`medium`**, **`high`** – より複雑な推論のために段階的に熟考時間を増加。

Anthropicモデルは `off` がデフォルト、GPT-5は `medium` から開始します。

### 自律レベル

`autonomyMode` はセッション開始時にdroidがコマンドをどの程度積極的に実行するかを制御します。`normal` から始めるか、`auto-*` プリセットを選択して追加のアクションを事前承認できます。

### Diffモード

droidがコード変更を表示する方法を制御：

- **`github`** – サイドバイサイド、高忠実度レンダリング（推奨）。
- **`unified`** – 従来の単一列diff形式。

### クラウドセッション同期

このスイッチがオンの場合、すべてのCLIセッションがFactory webにミラーされ、ブラウザで会話を再確認できます：

- **`true`** – セッションをwebアプリに同期。
- **`false`** – セッションをローカルのみに保持。

### 音声通知

droidイベントの音声フィードバックを設定：

**完了音**（`completionSound`） - 応答完了時に再生：
- **`fx-ok01`** – 内蔵完了音（デフォルト） - ソフトな成功音
- **`fx-ack01`** – 代替内蔵効果音 - 触覚的なリップルフィードバック
- **`bell`** – システムターミナルベルを使用
- **`off`** – 音声通知なし
- **カスタムパス** – 独自の音声ファイルへのパスを提供（例：`"/path/to/sound.wav"`）

**入力待機音**（`awaitingInputSound`） - droidがユーザー入力を待機中に再生。完了音と同じオプション、デフォルトは `fx-ack01`。

**音声フォーカスモード**（`soundFocusMode`） - 音声再生タイミングを制御：
- **`always`** – ウィンドウフォーカスに関係なく音声を再生（デフォルト）
- **`focused`** – ターミナルがフォーカスされている時のみ音声を再生
- **`unfocused`** – ターミナルがフォーカスされていない時のみ音声を再生

<Note>
  音声設定には `/settings` またはTUIで `Shift+Tab` → **Settings** からアクセスできます。
</Note>

### フック

`hooksDisabled` 設定は、フック設定を削除することなくすべてのフック実行を無効化するグローバルトグルを提供します：

- **`false`** – フックが有効で正常に実行される（デフォルト）
- **`true`** – すべてのフックがグローバルで無効化

これは `/hooks` メニューまたは `/settings` からもトグルできます。

### IDE自動接続

`ideAutoConnect` 設定は、外部ターミナル（IDEの内蔵ターミナル以外）から実行する際にdroidがIDEに自動接続するかを制御します：

- **`false`** – IDEターミナル内で実行時のみ自動接続（デフォルト）
- **`true`** – 任意のターミナルからIDEに自動接続

### Todo表示モード

`todoDisplayMode` 設定は、UIでのTodoリストの表示方法を制御します：

- **`pinned`** – Todoリストが入力エリア上に固定表示（デフォルト）
- **`inline`** – Todoリストがメッセージフロー内にインライン表示

## コマンドの許可リストと拒否リスト

これらの設定を使用して、droidが自動実行できるコマンドと実行してはいけないコマンドを制御：

- **`commandAllowlist`** – この配列のコマンドは安全として扱われ、自律プロンプトに関係なく追加確認なしで実行されます。頻繁に使用する低リスクのユーティリティのみを含めてください（例：`ls`、`pwd`、`dir`）。
- **`commandDenylist`** – この配列のコマンドは常に確認が必要で、破壊的または危険なために通常ブロックされます（例：再帰的な `rm`、`mkfs`、特権システム操作）。

両方のリストに現れるコマンドは拒否リストの動作がデフォルトになります。どちらのリストにもないコマンドは、セッションで選択した自律レベルにフォールバックします。

### 許可/拒否設定の例```json
{
  "commandAllowlist": [
    "ls",
    "pwd",
    "dir"
  ],
  "commandDenylist": [
    "rm -rf /",
    "mkfs",
    "shutdown"
  ]
}
```これらの配列を定期的に確認・更新して、ワークフローとセキュリティ体制に適合するようにしてください。特にチーム間で設定を共有する場合は重要です。

## 設定例```json
{
  "model": "opus",
  "reasoningEffort": "low",
  "diffMode": "github",
  "cloudSessionSync": true,
  "completionSound": "fx-ok01",
  "awaitingInputSound": "fx-ack01",
  "soundFocusMode": "always",
  "todoDisplayMode": "pinned"
}
```---
