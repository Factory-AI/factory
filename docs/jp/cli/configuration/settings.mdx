## 設定へのアクセス

droid設定を構成するには:

1. `droid`を実行
2. `/settings`を入力
3. 設定を対話的に調整

変更は即座に反映され、設定ファイルに保存されます。

## 設定の保存場所

| OS            | 場所                                   |
| ------------- | -------------------------------------- |
| macOS / Linux | `~/.factory/settings.json`             |
| Windows       | `%USERPROFILE%\.factory\settings.json` |

ファイルが存在しない場合、初回**droid**実行時にデフォルト値で作成されます。

### ローカルオーバーライド

任意の`.factory/`フォルダ内で、`settings.json`と並行して`settings.local.json`を作成できます:

- `~/.factory/settings.local.json` (ユーザーレベル)
- `<project>/.factory/settings.local.json` (プロジェクトレベル)

ローカルオーバーライドは同じレベルの対応する`settings.json`の上にマージされ、同じ階層の優先順位に従います。マシン固有の設定をバージョン管理から除外したい場合は、`settings.local.json`を`.gitignore`に追加してください。

### 推論努力

`reasoningEffort`は、モデルが応答前に実行する構造化された思考の量を調整します。利用可能な値はモデルによって異なりますが、一般的に以下が含まれます:

- **`off` / `none`** – 構造化された推論を無効化 (最高速)。
- **`low`、`medium`、`high`** – より複雑な推論のために段階的に熟慮時間を増加。

Anthropicモデルはデフォルトで`off`、GPT-5は`medium`から開始します。

### 自律レベル

`autonomyMode`は、セッション開始時にdroidがどの程度積極的にコマンドを実行するかを制御します。`normal`から始めるか、`auto-*`プリセットを選択して追加のアクションを事前承認します。

### 差分モード

droidのコード変更表示方法を制御します:

- **`github`** – サイドバイサイド、高忠実度レンダリング (推奨)。
- **`unified`** – 従来の単一カラム差分形式。

### クラウドセッション同期

このスイッチがオンの時、すべてのCLIセッションがFactory webにミラーリングされ、ブラウザで会話を再確認できます:

- **`true`** – セッションをwebアプリに同期。
- **`false`** – セッションをローカルのみに保持。

### サウンド通知

droidイベントのオーディオフィードバックを設定します:

**完了サウンド** (`completionSound`) - レスポンス完了時に再生:

- **`fx-ok01`** – 内蔵完了サウンド (デフォルト) - ソフトな成功音
- **`fx-ack01`** – 代替内蔵サウンドエフェクト - 触覚的なリップルフィードバック
- **`bell`** – システムターミナルベルを使用
- **`off`** – サウンド通知なし
- **カスタムパス** – 独自のサウンドファイルのパスを指定 (例: `"/path/to/sound.wav"`)

**入力待機サウンド** (`awaitingInputSound`) - droidがユーザー入力を待機中に再生。完了サウンドと同じオプション、デフォルトは`fx-ack01`。

**サウンドフォーカスモード** (`soundFocusMode`) - サウンド再生タイミングを制御:

- **`always`** – ウィンドウフォーカスに関係なくサウンドを再生 (デフォルト)
- **`focused`** – ターミナルがフォーカス中のみサウンドを再生
- **`unfocused`** – ターミナルがフォーカス外の時のみサウンドを再生

<Note>サウンド設定には`/settings`またはTUIで`Shift+Tab` → **Settings**からアクセスできます。</Note>

### フック

`hooksDisabled`設定は、フック設定を削除することなくすべてのフック実行を無効化するグローバルトグルを提供します:

- **`false`** – フックが有効で正常に実行されます (デフォルト)
- **`true`** – すべてのフックがグローバルに無効化されます

これは`/hooks`メニューや`/settings`からも切り替え可能です。

### IDE自動接続

`ideAutoConnect`設定は、外部ターミナル（IDE内蔵ターミナル以外）から実行する際にdroidが自動的にIDEに接続するかを制御します:

- **`false`** – IDEターミナル内で実行時のみ自動接続 (デフォルト)
- **`true`** – 任意のターミナルからIDEに自動接続

### TODO表示モード

`todoDisplayMode`設定は、UIでのTODOリストの表示方法を制御します:

- **`pinned`** – TODOリストが入力エリア上にピン留め (デフォルト)
- **`inline`** – TODOリストがメッセージフロー内にインライン表示

## コマンド許可リストと拒否リスト

これらの設定を使用して、droidが自動実行できるコマンドと実行してはいけないコマンドを制御します:

- **`commandAllowlist`** – この配列内のコマンドは安全として扱われ、自律性プロンプトに関係なく追加確認なしで実行されます。頻繁に使用する低リスクユーティリティのみを含めてください (例: `ls`、`pwd`、`dir`)。
- **`commandDenylist`** – この配列内のコマンドは常に確認が必要で、通常は破壊的または危険なため除外されます (例: 再帰的`rm`、`mkfs`、特権システム操作)。

両方のリストに表示されるコマンドは、デフォルトで拒否リストの動作になります。どちらのリストにもないコマンドは、セッションで選択した自律レベルにフォールバックします。

### 許可/拒否設定例```json
{
  "commandAllowlist": ["ls", "pwd", "dir"],
  "commandDenylist": ["rm -rf /", "mkfs", "shutdown"]
}
```定期的にこれらの配列をレビューし、特にチーム間で設定を共有する際には、ワークフローとセキュリティポスチャに合わせて更新してください。
