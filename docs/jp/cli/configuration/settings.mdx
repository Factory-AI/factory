## 設定へのアクセス

droidの設定を構成するには：

1. `droid`を実行
2. `/settings`を入力
3. 設定をインタラクティブに調整

変更は即座に有効になり、設定ファイルに保存されます。

## 設定の保存場所

| OS        | 場所                               |
| --------- | -------------------------------------- |
| macOS / Linux | `~/.factory/settings.json` |
| Windows   | `%USERPROFILE%\.factory\settings.json` |

ファイルが存在しない場合は、**droid**を初回実行時にデフォルト設定で作成されます。

### ローカルオーバーライド

任意の`.factory/`フォルダー内で`settings.json`と並列に`settings.local.json`を作成できます：

- `~/.factory/settings.local.json`（ユーザーレベル）
- `<project>/.factory/settings.local.json`（プロジェクトレベル）

ローカルオーバーライドは、同じレベルの対応する`settings.json`の上にマージされ、同じ階層優先順位に従います。マシン固有の設定をバージョン管理から除外したい場合は、`settings.local.json`を`.gitignore`に追加してください。

## 利用可能な設定

| 設定 | オプション | デフォルト | 説明 |
| ------- | ------- | ------- | ----------- |
| `model` | `opus`, `opus-4-6`, `opus-4-6-fast`, `sonnet`, `sonnet-4-6`, `gpt-5.4`, `gpt-5.2`, `gpt-5.2-codex`, `gpt-5.3-codex`, `haiku`, `gemini-3.1-pro`, `gemini-3-flash`, `droid-core`, `glm-5`, `kimi-k2.5`, `minimax-m2.5`, `custom-model` | `opus` | droidが使用するデフォルトのAIモデル |
| `reasoningEffort` | `off`, `none`, `low`, `medium`, `high`（利用可能性はモデルに依存） | モデル依存のデフォルト | モデルが実行する構造化思考の量を制御します。 |
| `autonomyMode` | `normal`, `spec`, `auto-low`, `auto-medium`, `auto-high` | `normal` | droid開始時のデフォルト自律モードを設定します。 |
| `cloudSessionSync` | `true`, `false` | `true` | CLIセッションをFactory webにミラーリングします。 |
| `diffMode` | `github`, `unified` | `github` | 分割されたGitHubスタイルの差分と単一カラムビューから選択します。 |
| `completionSound` | `off`, `bell`, `fx-ok01`, `fx-ack01`, またはカスタムファイルパス | `fx-ok01` | レスポンス完了時のオーディオキュー。 |
| `awaitingInputSound` | `off`, `bell`, `fx-ok01`, `fx-ack01`, またはカスタムファイルパス | `fx-ack01` | droidがユーザー入力を待機中のオーディオキュー。 |
| `soundFocusMode` | `always`, `focused`, `unfocused` | `always` | サウンド通知を再生するタイミング。 |
| `commandAllowlist` | コマンドの配列 | 安全なデフォルトを提供 | 追加確認なしで実行されるコマンド。 |
| `commandDenylist` | コマンドの配列 | 制限的なデフォルトを提供 | 常に確認が必要なコマンド。 |
| `includeCoAuthoredByDroid` | `true`, `false` | `true` | Droidの共同作成者トレーラーをコミットに自動追加します。 |
| `enableDroidShield` | `true`, `false` | `true` | シークレットスキャンとgitガードレールを有効にします。 |
| `hooksDisabled` | `true`, `false` | `false` | 全てのフック実行をグローバルに無効化します。 |
| `ideAutoConnect` | `true`, `false` | `false` | 外部ターミナルからIDEに自動接続します。 |
| `todoDisplayMode` | `inline`, `pinned` | `pinned` | UIでのTodoリストの表示方法。 |
| `showThinkingInMainView` | `true`, `false` | `false` | メインチャットビューでAI思考/推論ブロックを表示します。 |
| `customModels` | モデル設定の配列 | `[]` | BYOKのカスタムモデル設定。[BYOKドキュメント](/jp/cli/configuration/byok)を参照してください。 |

### リーズニングエフォート

`reasoningEffort`は、モデルが返答前に実行する構造化思考の量を調整します。利用可能な値はモデルに依存しますが、通常以下が含まれます：

- **`off` / `none`** – 構造化リーズニングを無効化（最速）。
- **`low`**, **`medium`**, **`high`** – より複雑なリーズニングのため熟考時間を段階的に増加。

Anthropicモデルはデフォルトで`off`、GPT-5は`medium`で開始されます。

### 自律レベル

`autonomyMode`は、セッション開始時にdroidがコマンドをどれだけ積極的に実行するかを制御します。`normal`から開始するか、`auto-*`プリセットを選択して追加アクションを事前承認できます。

### 差分モード

droidがコード変更を表示する方法を制御します：

- **`github`** – 横並び、高忠実度レンダー（推奨）。
- **`unified`** – 従来の単一カラム差分フォーマット。

### クラウドセッション同期

このスイッチがオンの場合、すべてのCLIセッションがFactory webにミラーリングされ、ブラウザーで会話を再訪できます：

- **`true`** – セッションをWebアプリに同期。
- **`false`** – セッションをローカルのみに保持。

### サウンド通知

droidイベントのオーディオフィードバックを設定します：

**完了サウンド**（`completionSound`） - レスポンス完了時に再生：
- **`fx-ok01`** – 内蔵完了サウンド（デフォルト） - ソフトな成功音
- **`fx-ack01`** – 代替内蔵サウンドエフェクト - 触覚的なリップルフィードバック
- **`bell`** – システムターミナルベルを使用
- **`off`** – サウンド通知なし
- **カスタムパス** – 独自のサウンドファイルへのファイルパスを提供（例：`"/path/to/sound.wav"`）

**入力待機サウンド**（`awaitingInputSound`） - droidがユーザー入力を待機中に再生。完了サウンドと同じオプション、デフォルトは`fx-ack01`。

**サウンドフォーカスモード**（`soundFocusMode`） - サウンドを再生するタイミングを制御：
- **`always`** – ウィンドウフォーカスに関係なくサウンドを再生（デフォルト）
- **`focused`** – ターミナルがフォーカスされている時のみサウンドを再生
- **`unfocused`** – ターミナルがフォーカスされていない時のみサウンドを再生

<Note>
  TUIで`/settings`または`Shift+Tab` → **Settings**からサウンド設定にアクセスしてください。
</Note>

### フック

`hooksDisabled`設定は、フック設定を削除せずにすべてのフック実行を無効にするグローバルトグルを提供します：

- **`false`** – フックが有効化され、正常に実行されます（デフォルト）
- **`true`** – 全てのフックがグローバルに無効化されます

`/hooks`メニューまたは`/settings`からもこれを切り替えできます。

### IDE自動接続

`ideAutoConnect`設定は、外部ターミナル（IDEの内蔵ターミナル外）から実行時にdroidが自動的にIDEに接続するかを制御します：

- **`false`** – IDEターミナル内で実行時のみ自動接続（デフォルト）
- **`true`** – 任意のターミナルからIDEに自動接続

### Todo表示モード

`todoDisplayMode`設定は、UIでのTodoリストの表示方法を制御します：

- **`pinned`** – Todoリストが入力エリアの上に固定表示（デフォルト）
- **`inline`** – Todoリストがメッセージフロー内にインライン表示

## コマンド許可リストと拒否リスト

これらの設定を使用して、droidが自動実行できるコマンドと実行してはならないコマンドを制御します：

- **`commandAllowlist`** – この配列内のコマンドは安全として扱われ、自律プロンプトに関係なく追加確認なしで実行されます。頻繁に使用する低リスクのユーティリティのみを含めてください（例：`ls`、`pwd`、`dir`）。
- **`commandDenylist`** – この配列内のコマンドは常に確認が必要で、破壊的または危険なため通常ブロックされます（例：再帰的`rm`、`mkfs`、特権システム操作）。

両方のリストに表示されるコマンドは、拒否リストの動作がデフォルトとなります。どちらのリストにもないコマンドは、セッションで選択した自律レベルにフォールバックします。

### 許可/拒否設定の例```json
{
  "commandAllowlist": [
    "ls",
    "pwd",
    "dir"
  ],
  "commandDenylist": [
    "rm -rf /",
    "mkfs",
    "shutdown"
  ]
}
```これらの配列は、特にチーム間で設定を共有する場合、ワークフローとセキュリティ体制に合わせて定期的に見直しと更新を行ってください。

{
  "model": "opus",

  "diffMode": "github",
  "cloudSessionSync": true,
  "completionSound": "fx-ok01",

  "soundFocusMode": "always",
  "todoDisplayMode": "pinned"
}
```---
