Droid Computer は、Droid が複数のセッションにわたって接続し作業できる永続的なコンピュート環境です。各セッション後に破棄される一時的なクラウドテンプレートとは異なり、コンピューターは**状態を保持**します。インストールされたパッケージ、ファイル、実行中のサービス、設定はすべてセッション間で持続されます。

## コンピューターのタイプ

Factory では Droid Computer を使用する2つの方法をサポートしています：

- **マネージドコンピューター** — Factory があなたのために基盤となるクラウドコンピューターをプロビジョニングし管理します
- **Bring Your Own Machine (BYOM)** — VPS、ワークステーション、オンプレミスサーバーなど、あなたが既に管理しているマシンを登録します

両方のタイプは **設定 → Droid Computer** に表示され、Droid セッションの長期実行ターゲットとして使用できます。

## セッションでのコンピューターの使用

### ウェブアプリ

新規セッションモーダルで、**コンピューター** タブを選択し、アクティブなコンピューターを選択し、作業ディレクトリ（例：`/home/factory-user/projects/my-app`）を設定してセッションを開始します。

接続ステータス（接続中 / 接続済み / エラー）がリアルタイムで表示されます。

### Slack

コンピューターは Slack 連携を通じてセッションを作成する際にも選択できます。

## コンピューターの管理

**設定 → Droid Computer** から、次のことができます：

- 自分のローカルコンピューターを設定する
- 利用可能なコンピューターを閲覧する
- コンピューターの名前を変更する
- コンピューターを削除する
- ステータスと接続情報の詳細ページを開く（マネージドコンピューターのみ）

## Bring Your Own Machine (BYOM)

VPS、クラウド VM、オンプレミスサーバーなど、あなたが所有する任意のマシンを Droid Computer として登録できます。完全な BYOM セットアップフロー、リモートアクセスガイダンス、管理ノートについては、[Bring Your Own Machine (BYOM)](/jp/cli/features/droid-computers-byom) を参照してください。

## マネージドコンピューター

<Note>
**研究プレビュー** — マネージドコンピューターは現在、選択された組織でのみ利用可能です。
</Note>

最も簡単に始める方法は、Factory ウェブアプリから直接コンピューターを作成することです。

1. **設定 → Droid Computer** に移動します。
2. **作成** をクリックします。
3. コンピューターに名前を付けます。

Factory がクラウドコンピューター（4 CPU、8GB RAM、6GB スワップ）を自動的にプロビジョニングします。プロビジョニングの進行状況は、以下のステップを通じて UI でライブトラッキングされます：

1. **コンピューターを作成中** — クラウドコンピューターを割り当て中
2. **ユーザーをセットアップ中** — sudo アクセス権を持つ `factory-user` アカウントを作成中
3. **環境を設定中** — 環境設定、SSH キー、サービスファイルを書き込み中
4. **Droid をインストール中** — Droid バイナリをダウンロードしてインストール中
5. **サービスを開始中** — SSH と Droid デーモンサービスを起動中

プロビジョニングが完了すると、コンピューターのステータスが **アクティブ** に変わり、セッションで使用できるようになります。プラットフォーム管理のコンピューターは、アイドル時に自動一時停止し、新しいセッションがそれらをターゲットにすると自動再開します。また、プロビジョニングの詳細、リソースメトリクス、リモート更新フローも公開されます。

## CLI コマンド

### 一般

| コマンド | 説明 |
|---------|------|
| `droid computer list` | すべてのコンピューターをリスト表示（名前、ID、ステータスを表示；現在のマシンをマーク） |
| `droid computer ssh <name>` | 任意のコンピューターへのインタラクティブ SSH セッションを開く |

### BYOM セットアップ

| コマンド | 説明 |
|---------|------|
| `droid computer register [name]` | 現在のマシンを BYOM コンピューターとして登録 |
| `droid computer remove` | 現在のマシンの登録を解除し、ローカル設定をクリーンアップ |

### SSH オプション

`ssh` サブコマンドは以下のフラグをサポートします：

- `--proxy` — stdio プロキシとして実行（`ProxyCommand` として使用）
- `--port <port>` — リモートマシンのターゲットポート（デフォルト：22）
- `--debug` — 詳細な接続ログを有効化

## 監視

**設定 → Droid Computer** で任意のコンピューターをクリックして詳細ページを開きます。プラットフォーム管理のコンピューターの場合、詳細ページはライブリソースメトリクスを表示します：

- **CPU 使用率** — 時間経過による使用率パーセンテージ
- **メモリ使用量** — 使用メモリ対総メモリ
- **ディスク使用量** — 使用ディスク対総ディスク容量

詳細ページには現在の**デーモンバージョン**と接続ステータスも表示されます。可能なコンピューターステータスは：

- **プロビジョニング中** — 初期セットアップが進行中
- **アクティブ** — セッション準備完了
- **エラー** — プロビジョニングまたはランタイム障害

## デーモンの更新

コンピューター詳細ページには、実行中のデーモンバージョンと利用可能な最新バージョンが表示されます。更新が利用可能な場合、**更新** ボタンが表示されます。クリックするとリモート更新がトリガーされ、デーモンが新しいバイナリをダウンロードし、再起動し、自動的に再接続します。

## Git 認証情報

- **BYOM** — Droid はマシンに既に設定されている git 認証情報を使用します。
- **マネージドコンピューター** — Factory との個人 GitHub 連携がある場合、認証されたリポジトリアクセス用の認証情報が自動的に追加されます。新しい GitHub App インストールを追加したり、リポジトリアクセスを変更したりすると、次のセッション接続時に認証情報が自動的に更新されます。

## SSH & IDE 連携

`droid computer ssh <name>` はデーモンを通じて安全な WebSocket トンネルを確立します。直接 SSH ポートを公開する必要はありません。Factory は専用の Ed25519 SSH キーペア（`~/.factory/.ssh/` に保存）を生成し管理し、個人の SSH キーとは分離されています。公開キーは各接続時にパスワードなし認証のために注入されます。

### VS Code Remote-SSH

VS Code Remote-SSH の `ProxyCommand` として `droid computer ssh` を使用できます：```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  IdentitiesOnly yes
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```これにより、Droid Computer上でフルのVS Codeリモートセッションを開くことができます。

## セキュリティ

- **ファイアウォール** — プラットフォーム管理されたコンピューターは、iptablesルールを使用してデーモンポートのみへのインバウンドトラフィックを制限します。他のすべてのポートはデフォルトでブロックされます。なお、`factory-user`にはパスワードなしのsudoアクセス権があり、これらのルールを変更できます。厳密なネットワークレベルの境界が必要な場合は、インフラストラクチャレベルですべてのパブリックトラフィックをブロックするリレーモードを使用してください。
- **SSHの強化** — rootログインとパスワード認証は無効化されています。
- **リレーモード** — BYOMコンピューターはすべてのトラフィックをFactoryのリレーサービス経由でルーティングするため、パブリックポートは公開されません。プラットフォーム管理されたコンピューターも、より厳格なネットワーク分離のために組織レベルでリレーモードを使用するように設定できます。
- **Git認証情報** — プラットフォーム管理されたコンピューター上で、認証されたリポジトリアクセスのためにユーザーごとに自動的に設定されます。
