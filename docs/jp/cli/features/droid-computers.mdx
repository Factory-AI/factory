<Note>
**リサーチプレビュー** — Droid Computersは現在、選ばれた組織のみが利用できます。**ユーザーあたり1台のコンピューター**の制限があります。
</Note>

Droid Computersは、Droidがセッション間で接続して作業できる永続的なコンピューティング環境です。各セッション後に削除される一時的なクラウドテンプレートとは異なり、コンピューターは**状態を保持**します — インストールされたパッケージ、ファイル、実行中のサービス、および構成はすべてセッション間で持続します。

## プラットフォーム管理のコンピューター

開始する最も早い方法は、Factory Web アプリから直接コンピューターを作成することです。

1. **Settings → Droid Computers**に移動します。
2. **Create**をクリックします。
3. コンピューターに名前を付けます。

Factoryは自動的にクラウドコンピューター（4 CPU、8GB RAM、6GBスワップ）をプロビジョニングします。プロビジョニングの進行状況は、以下のステップを通じてUI内でリアルタイムに追跡されます：

1. **Creating computer** — クラウドコンピューターの割り当て
2. **Setting up user** — sudo権限を持つ`factory-user`アカウントの作成
3. **Configuring environment** — 環境設定、SSHキー、サービスファイルの書き込み
4. **Installing Droid** — Droidバイナリのダウンロードとインストール
5. **Starting services** — SSHおよびDroidデーモンサービスの起動

いずれかのステップが失敗した場合、コンピューターのステータスが**Error**に変わり、失敗したステップがUIに表示されます。コンピューターを削除して再試行できます。

プロビジョニングが完了すると、コンピューターのステータスが**Active**に変わり、セッションで使用可能になります。プラットフォーム管理のコンピューターは、アイドル時に自動一時停止し、新しいセッションがそれらをターゲットにしたときに自動再開します。

## コンピューターの管理

**Settings → Droid Computers**のコンピューター詳細ページから、以下を実行できます：

- コンピューターの**名前変更**
- コンピューターの**削除**（これにより永続的に停止・削除されます）
- ライブリソースメトリクスとデーモンバージョンの表示（[監視](#monitoring)を参照）

## Bring Your Own Machine (BYOM)

VPS、クラウドVM、オンプレミスサーバーなど、任意のマシンをDroid Computerとして登録できます。

### 前提条件

- **Linux**または**macOS**を実行するマシン
- マシンにDroid CLIがインストールされていること
- Factoryで認証済み（インタラクティブセッションで`/login`）
- Factory APIへのネットワークアクセス、特にrelay.factory.ai（インバウンドポートを開く必要はありません）

### セットアップ

マシンを登録します：```bash
droid computer register [name]
```それから Factory がマシンに接続できるようにデーモンを開始してください：```bash
droid daemon --remote-access
````--remote-access` フラグは、デーモンがFactoryのセキュアリレーサービス経由で接続するように指示するため、マシンでインバウンドポートを開く必要がありません。

<Note>
一度に1台のマシンにつき1つのコンピュータ登録のみ許可されます。再登録が必要な場合は、まず `droid computer remove` を実行するか、設定ページからマシンを削除してください。
</Note>

## CLI コマンド

### 全般

| コマンド | 説明 |
|---------|-------------|
| `droid computer list` | すべてのコンピュータを一覧表示（名前、ID、ステータスを表示、現在のマシンをマーク） |
| `droid computer ssh <name>` | 任意のコンピュータへの対話型SSH セッションを開始 |

### BYOM セットアップ

| コマンド | 説明 |
|---------|-------------|
| `droid computer register [name]` | 現在のマシンをBYOM コンピュータとして登録 |
| `droid computer remove` | 現在のマシンの登録を解除し、ローカル設定をクリーンアップ |

### SSH オプション

`ssh` サブコマンドは以下のフラグをサポートします：

- `--proxy` — stdio プロキシとして実行（`ProxyCommand` として使用）
- `--port <port>` — リモートマシンのターゲットポート（デフォルト: 22）
- `--debug` — 詳細な接続ログを有効化

## セッションでのコンピュータの使用

### Web アプリ

新しいセッションモーダルで、**コンピュータ** タブを選択し、アクティブなコンピュータを選び、作業ディレクトリを設定して（例：`/home/factory-user/projects/my-app`）、セッションを開始します。

接続ステータス（接続中 / 接続済み / エラー）がリアルタイムで表示されます。

### Slack

Slack 連携経由でセッションを作成する際にもコンピュータを選択できます。

## 監視

**設定 → Droid コンピュータ** の任意のコンピュータをクリックして詳細ページを開きます。プラットフォーム管理のコンピュータの場合、詳細ページにはライブリソースメトリクスが表示されます：

- **CPU 使用率** — 時間経過に伴う使用率の割合
- **メモリ使用量** — 使用済み対総メモリ
- **ディスク使用量** — 使用済み対総ディスク容量

詳細ページには現在の **デーモンバージョン** と接続ステータスも表示されます。コンピュータステータスは以下の通りです：

- **プロビジョニング中** — 初期セットアップ進行中
- **アクティブ** — セッション準備完了
- **エラー** — プロビジョニングまたはランタイム障害

## デーモンの更新

コンピュータ詳細ページには、実行中のデーモンバージョンと利用可能な最新バージョンが表示されます。更新が利用可能な場合、**更新** ボタンが表示されます。これをクリックするとリモート更新がトリガーされ、デーモンが新しいバイナリをダウンロードし、再起動して自動的に再接続します。

BYOM マシンの場合は、最新版をダウンロードしてデーモンプロセスを再起動することで、手動でDroid CLI を更新することもできます。

## Git 認証情報

プラットフォーム管理のコンピュータでは、認証されたリポジトリアクセスのために Git 認証情報が自動的に設定されます。新しい GitHub App インストールを追加したり、リポジトリアクセスを変更したりすると、次回のセッション接続時に git 認証情報が自動的に更新されます。

BYOM コンピュータは、マシンに既に設定されている git 認証情報を使用します。

## SSH & IDE 連携

`droid computer ssh <name>` は、デーモン経由でセキュアな WebSocket トンネルを確立します。直接的な SSH ポートの公開は必要ありません。Factory は専用の Ed25519 SSH キーペア（`~/.factory/.ssh/` に格納）を生成・管理し、個人の SSH キーとは分離されています。公開キーは各接続時にパスワードレス認証のために注入されます。

### VS Code Remote-SSH

`droid computer ssh` を VS Code Remote-SSH の `ProxyCommand` として使用できます：```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  IdentitiesOnly yes
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```これにより、Droid Computer上でフル VS Code リモートセッションを開くことができます。

<Note>
プロキシモードは、コンピュータを解決する際にホスト名から `factory-` プレフィックスを自動的に除去します。例えば、`factory-mycomputer` という名前のホストエントリは、`mycomputer` という名前のコンピュータに接続します。
</Note>

## セキュリティ

- **ファイアウォール** — プラットフォーム管理コンピュータは、iptables ルールを使用してデーモンポートのみへのインバウンドトラフィックを制限し、その他すべてのポートはデフォルトでブロックされます。なお、`factory-user` はパスワードなしの sudo アクセス権を持ち、これらのルールを変更できることに注意してください。ハードなネットワークレベルの境界が必要な場合は、インフラストラクチャレベルですべてのパブリックトラフィックをブロックするリレーモードを使用してください。
- **SSH 強化** — ルートログインとパスワード認証は無効化されています。
- **リレーモード** — BYOM コンピュータはすべてのトラフィックを Factory のリレーサービス経由でルーティングするため、パブリックポートが公開されることはありません。プラットフォーム管理コンピュータも、より厳格なネットワーク分離のため、組織レベルでリレーモードを使用するように設定できます。
- **Git 認証情報** — プラットフォーム管理コンピュータでは、認証されたリポジトリアクセスのためにユーザーごとに自動的に設定されます。
