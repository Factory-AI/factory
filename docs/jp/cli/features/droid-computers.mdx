<Note>
**研究プレビュー** — Droid Computersは現在、選択された組織でのみ利用可能です。**ユーザーあたり1台のコンピューター**という制限があります。
</Note>

Droid Computersは、Droidが接続してセッション間で作業を継続できる永続的なコンピューティング環境です。各セッション後に破棄される一時的なクラウドテンプレートとは異なり、コンピューターは**状態を保持**します — インストール済みパッケージ、ファイル、実行中のサービス、設定はすべてセッション間で保持されます。

## プラットフォーム管理コンピューター

最も簡単に始める方法は、Factoryウェブアプリからコンピューターを直接作成することです。

1. **設定 → Droid Computers**に移動します。
2. **作成**をクリックします。
3. コンピューターに名前を付けます。

Factoryがクラウドコンピューター（4 CPU、8GB RAM、6GB swap）を自動的にプロビジョニングします。プロビジョニングの進行状況は、以下のステップを通してUIでリアルタイムに追跡されます：

1. **コンピューターの作成** — クラウドコンピューターの割り当て
2. **ユーザーの設定** — sudo権限を持つ`factory-user`アカウントの作成
3. **環境の設定** — 環境設定、SSHキー、サービスファイルの書き込み
4. **Droidのインストール** — Droidバイナリのダウンロードとインストール
5. **サービスの開始** — SSHとDroidデーモンサービスの起動

いずれかのステップが失敗した場合、コンピューターのステータスは**エラー**に変わり、失敗したステップがUIに表示されます。コンピューターを削除して再試行できます。

プロビジョニングが完了すると、コンピューターのステータスは**アクティブ**に変わり、セッションで使用できるようになります。プラットフォーム管理コンピューターは、アイドル時に自動一時停止し、新しいセッションがそれらを対象とすると自動再開します。

## コンピューターの管理

**設定 → Droid Computers**のコンピューター詳細ページから、以下のことができます：

- コンピューターの**名前変更**
- コンピューターの**削除**（これにより永続的に停止・削除されます）
- リアルタイムリソースメトリクスとデーモンバージョンの表示（[監視](#monitoring)を参照）

## Bring Your Own Machine (BYOM)

任意のマシン — VPS、クラウドVM、オンプレミスサーバーなど — をDroid Computerとして登録できます。

### 前提条件

- **Linux**、**macOS**、または**Windows**を実行するマシン
- マシンにインストールされたDroid CLI
- Factoryで認証済み（インタラクティブセッションで`/login`）
- Factory APIへのネットワークアクセス、特にrelay.factory.ai（インバウンドポートを開く必要はありません）

### セットアップ

マシンを登録します：```bash
droid computer register [name]
```その後、Factoryがマシンに接続できるようにデーモンを起動します：```bash
droid daemon --remote-access
````--remote-access` フラグは、デーモンにFactoryの安全なリレーサービス経由で接続するよう指示するため、マシン上でインバウンドポートを開く必要がありません。

<Note>
一度に許可されるコンピュータ登録は、マシンごとに1つまでです。再登録が必要な場合は、まず `droid computer remove` を実行するか、設定ページからマシンを削除してください。
</Note>

## CLIコマンド

### 全般

| コマンド | 説明 |
|---------|-------------|
| `droid computer list` | すべてのコンピュータを一覧表示（名前、ID、ステータスを表示し、現在のマシンをマーク） |
| `droid computer ssh <name>` | 任意のコンピュータへのインタラクティブSSHセッションを開く |

### BYOM設定

| コマンド | 説明 |
|---------|-------------|
| `droid computer register [name]` | 現在のマシンをBYOMコンピュータとして登録 |
| `droid computer remove` | 現在のマシンの登録を解除し、ローカル設定をクリーンアップ |

### SSHオプション

`ssh` サブコマンドは以下のフラグをサポートします：

- `--proxy` — stdioプロキシとして実行（`ProxyCommand` で使用）
- `--port <port>` — リモートマシンの対象ポート（デフォルト: 22）
- `--debug` — 詳細な接続ログを有効化

## セッションでのコンピュータ使用

### Webアプリ

新しいセッションモーダルで、**Computer**タブを選択し、アクティブなコンピュータを選択し、作業ディレクトリ（例：`/home/factory-user/projects/my-app`）を設定して、セッションを開始します。

接続ステータス（接続中 / 接続済み / エラー）がリアルタイムで表示されます。

### Slack

Slack統合を介してセッションを作成する際にも、コンピュータを選択できます。

## モニタリング

**設定 → Droid Computers** の任意のコンピュータをクリックして、詳細ページを開きます。プラットフォーム管理のコンピュータの場合、詳細ページではライブのリソースメトリクスが表示されます：

- **CPU使用率** — 時系列での使用率パーセンテージ
- **メモリ使用量** — 使用中 vs 総メモリ
- **ディスク使用量** — 使用中 vs 総ディスク容量

詳細ページには、現在の**デーモンバージョン**と接続ステータスも表示されます。可能なコンピュータステータスは：

- **Provisioning** — 初期設定が進行中
- **Active** — セッションの準備完了
- **Error** — プロビジョニングまたはランタイムの障害

## デーモンの更新

コンピュータの詳細ページには、利用可能な最新バージョンと並んで実行中のデーモンバージョンが表示されます。更新が利用可能な場合、**Update**ボタンが表示されます。これをクリックするとリモート更新がトリガーされ、デーモンが新しいバイナリをダウンロードし、再起動して自動的に再接続します。

BYOMマシンの場合、最新バージョンをダウンロードしてデーモンプロセスを再起動することで、Droid CLIを手動で更新することもできます。

## Git認証情報

プラットフォーム管理のコンピュータでは、認証されたリポジトリアクセス用にGit認証情報が自動的に設定されます。新しいGitHub Appインストールを追加したり、リポジトリアクセスを変更したりすると、次のセッション接続時にgit認証情報が自動的に更新されます。

BYOMコンピュータは、マシンに既に設定されているgit認証情報を使用します。

## SSH & IDE統合

`droid computer ssh <name>` は、デーモン経由で安全なWebSocketトンネルを確立します。直接的なSSHポート露出は必要ありません。Factoryは専用のEd25519 SSHキーペア（`~/.factory/.ssh/`に保存）を生成・管理し、個人のSSHキーとは分離します。公開キーは各接続時にパスワードなし認証のために注入されます。

### VS Code Remote-SSH

VS Code Remote-SSHの`ProxyCommand`として`droid computer ssh`を使用できます：```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  IdentitiesOnly yes
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```これにより、Droid Computer で完全な VS Code リモートセッションを開くことができます。

<Note>
プロキシモードは、コンピュータを解決する際に Host 名から `factory-` プレフィックスを自動的に削除します。例えば、`factory-mycomputer` という名前の Host エントリは、`mycomputer` という名前のコンピュータに接続されます。
</Note>

## セキュリティ

- **ファイアウォール** — プラットフォーム管理コンピュータは iptables ルールを使用して、デーモンポートのみへのインバウンドトラフィックを制限します。その他すべてのポートはデフォルトでブロックされます。なお、`factory-user` はパスワード不要の sudo アクセス権を持っており、これらのルールを変更することができます。ハードなネットワークレベルの境界が必要な場合は、インフラストラクチャレベルですべてのパブリックトラフィックをブロックするリレーモードを使用してください。
- **SSH セキュリティ強化** — ルートログインとパスワード認証は無効化されています。
- **リレーモード** — BYOM コンピュータはすべてのトラフィックを Factory のリレーサービス経由でルーティングするため、パブリックポートは公開されません。プラットフォーム管理コンピュータも、より厳格なネットワーク分離のために組織レベルでリレーモードを使用するよう設定できます。
- **Git 認証情報** — プラットフォーム管理コンピュータにおいて、認証されたリポジトリアクセスのためにユーザーごとに自動的に設定されます。
