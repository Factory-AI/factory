Droid Computerは、Droidがセッション間で接続して作業できる永続的なコンピューティング環境です。各セッション後に破棄される一時的なクラウドテンプレートとは異なり、コンピュータは**状態を保持**します — インストールされたパッケージ、ファイル、実行中のサービス、設定はすべてセッション間で永続化されます。

## コンピュータの種類

Factoryは、Droid Computerを使用する2つの方法をサポートしています：

- **Bring Your Own Machine (BYOM)** — VPS、ワークステーション、オンプレミスサーバーなど、すでに管理している機器を登録
- **マネージドコンピュータ** — Factoryが基盤となるクラウドコンピュータをプロビジョニングして管理

どちらのタイプも**設定 → Droid Computers**に表示され、Droidセッションの長期稼働ターゲットとして使用できます。

## セッションでのコンピュータ使用

### Webアプリ

新しいセッションモーダルで、**Computer**タブを選択し、アクティブなコンピュータを選択して、作業ディレクトリ（例：`/home/factory-user/projects/my-app`）を設定し、セッションを開始します。

接続ステータス（接続中 / 接続済み / エラー）がリアルタイムで表示されます。

### Slack

Slack統合でセッションを作成する際にもコンピュータを選択できます。

## コンピュータ管理

**設定 → Droid Computers**から以下のことができます：

- ローカルコンピュータの設定
- 利用可能なコンピュータの参照
- コンピュータの名前変更
- コンピュータの削除
- ステータスと接続情報の詳細ページを開く（マネージドコンピュータのみ）

## Bring Your Own Machine (BYOM)

VPS、クラウドVM、オンプレミスサーバーなど、所有する任意の機器をDroid Computerとして登録できます。BYOM設定フローの詳細、リモートアクセスのガイダンス、管理に関する注意事項については、[Bring Your Own Machine (BYOM)](/jp/cli/features/droid-computers-byom)をご覧ください。

## マネージドコンピュータ

<Note>
**リサーチプレビュー** — マネージドコンピュータは現在、選択された組織でのみ利用可能です。
</Note>

最も迅速に開始する方法は、Factory Webアプリからコンピュータを直接作成することです。

1. **設定 → Droid Computers**に移動します。
2. **作成**をクリックします。
3. コンピュータに名前を付けます。

Factoryが自動的にクラウドコンピュータ（4 CPU、8GB RAM、6GBスワップ）をプロビジョニングします。プロビジョニングの進行状況は、以下のステップでUIでライブトラッキングされます：

1. **コンピュータ作成中** — クラウドコンピュータの割り当て
2. **ユーザー設定中** — sudo権限を持つ`factory-user`アカウントの作成
3. **環境設定中** — 環境設定、SSHキー、サービスファイルの書き込み
4. **Droidインストール中** — Droidバイナリのダウンロードとインストール
5. **サービス開始中** — SSHとDroidデーモンサービスの起動

プロビジョニングが完了すると、コンピュータのステータスが**アクティブ**に変わり、セッションで使用可能になります。プラットフォーム管理のコンピュータは、アイドル時に自動一時停止し、新しいセッションがターゲットにすると自動再開します。プロビジョニングの詳細、リソースメトリクス、リモート更新フローも公開されます。

## CLIコマンド

### 一般

| コマンド | 説明 |
|---------|-------------|
| `droid computer list` | すべてのコンピュータを一覧表示（名前、ID、ステータスを表示し、現在の機器をマーク） |
| `droid computer ssh <name>` | 任意のコンピュータへのインタラクティブSSHセッションを開く |

### BYOM設定

| コマンド | 説明 |
|---------|-------------|
| `droid computer register [name]` | 現在の機器をBYOMコンピュータとして登録 |
| `droid computer remove` | 現在の機器を登録解除し、ローカル設定をクリーンアップ |

### SSHオプション

`ssh`サブコマンドは以下のフラグをサポートします：

- `--proxy` — stdioプロキシとして実行（`ProxyCommand`として使用）
- `--port <port>` — リモート機器のターゲットポート（デフォルト：22）
- `--debug` — 詳細な接続ログを有効化

## 監視

**設定 → Droid Computers**で任意のコンピュータをクリックして詳細ページを開きます。プラットフォーム管理のコンピュータの場合、詳細ページにライブリソースメトリクスが表示されます：

- **CPU使用率** — 時間経過に伴う使用率（パーセント）
- **メモリ使用量** — 使用中 vs 総メモリ
- **ディスク使用量** — 使用中 vs 総ディスク容量

詳細ページには現在の**デーモンバージョン**と接続ステータスも表示されます。可能なコンピュータステータスは次のとおりです：

- **プロビジョニング中** — 初期セットアップが進行中
- **アクティブ** — セッションの準備完了
- **エラー** — プロビジョニングまたは実行時の失敗

## デーモンの更新

コンピュータ詳細ページには、実行中のデーモンバージョンと利用可能な最新バージョンが表示されます。更新が利用可能な場合、**更新**ボタンが表示されます。クリックするとリモート更新がトリガーされ — デーモンが新しいバイナリをダウンロードし、再起動し、自動的に再接続します。

## Git認証情報

- **BYOM** — Droidは機器上ですでに設定されているgit認証情報を使用します。
- **マネージドコンピュータ** — FactoryでGitHubの個人統合がある場合、認証されたリポジトリアクセス用の認証情報が自動的に追加されます。新しいGitHub Appインストールを追加したり、リポジトリアクセスを変更したりすると、次のセッション接続時に認証情報が自動的に更新されます。

## SSH & IDE統合

`droid computer ssh <name>`は、デーモンを介してセキュアなWebSocketトンネルを確立します — 直接のSSHポートの公開は不要です。Factoryは専用のEd25519 SSHキーペア（`~/.factory/.ssh/`に保存）を生成・管理し、個人のSSHキーとは分離されています。公開キーは各接続時にパスワードなし認証のために注入されます。

### VS Code Remote-SSH

VS Code Remote-SSHの`ProxyCommand`として`droid computer ssh`を使用できます：```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  IdentitiesOnly yes
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```これにより、Droid Computer上でフルのVS Code リモートセッションを開くことができます。

## セキュリティ

- **ファイアウォール** — プラットフォーム管理コンピューターはiptablesルールを使用して、デーモンポートのみへの受信トラフィックを制限します。その他すべてのポートはデフォルトでブロックされます。なお、`factory-user`にはパスワードなしsudoアクセス権があり、これらのルールを変更できることに注意してください。ハードなネットワークレベルの境界が必要な場合は、インフラストラクチャレベルですべてのパブリックトラフィックをブロックするリレーモードを使用してください。
- **SSH強化** — ルートログインとパスワード認証は無効になっています。
- **リレーモード** — BYOMコンピューターはすべてのトラフィックをFactory のリレーサービス経由でルーティングするため、パブリックポートが公開されることはありません。プラットフォーム管理コンピューターも、より厳格なネットワーク分離のために組織レベルでリレーモードを使用するよう設定できます。
- **Git認証情報** — プラットフォーム管理コンピューター上で認証されたリポジトリアクセスのため、ユーザーごとに自動的に設定されます。
