Droid Computersは、Droidがセッション間で接続して作業を行える永続的なコンピュート環境です。各セッション後に破棄される一時的なクラウドテンプレートとは異なり、コンピュータは**状態を保持**します — インストール済みパッケージ、ファイル、実行中のサービス、設定がすべてセッション間で永続化されます。

## コンピュータタイプ

FactoryはDroid Computersを使用する2つの方法をサポートしています：

- **マネージドコンピュータ** — Factoryが基盤となるクラウドコンピュータをプロビジョニングし管理します
- **Bring Your Own Machine (BYOM)** — VPS、ワークステーション、オンプレミスサーバーなど、すでに管理している機器を登録します

両方のタイプは**Settings → Droid Computers**に表示され、Droidセッションの長期的なターゲットとして使用できます。

## セッションでのコンピュータの使用

### Webアプリ

新しいセッションモーダルで、**Computer**タブを選択し、アクティブなコンピュータを選択して、作業ディレクトリ（例：`/home/factory-user/projects/my-app`）を設定し、セッションを開始します。

接続ステータス（接続中 / 接続済み / エラー）がリアルタイムで表示されます。

### Slack

Slack連携でセッションを作成する際にもコンピュータを選択できます。

## コンピュータの管理

**Settings → Droid Computers**から、以下の操作を行えます：

- 自分のローカルコンピュータの設定
- 利用可能なコンピュータの参照
- コンピュータの名前変更
- コンピュータの削除
- ステータスと接続情報の詳細ページを開く

## Bring Your Own Machine (BYOM)

任意の機器（VPS、クラウドVM、オンプレミスサーバーなど）をDroid Computerとして登録できます。

- サポートOS：**Linux**、**macOS**、**Windows**
- セットアップは`droid computer register [name]`と`droid daemon --remote-access`を使用
- トラフィックはFactoryのリレーサービス経由でルーティングされるため、インバウンドポートを開く必要はありません

完全なBYOMセットアップフロー、リモートアクセスガイダンス、管理ノートについては、[Bring Your Own Machine (BYOM)](/jp/cli/features/droid-computers-byom)を参照してください。

## マネージドコンピュータ

<Note>
**研究プレビュー** — マネージドコンピュータは現在、選択された組織のみが利用可能です。
</Note>

最も迅速に開始する方法は、Factory Webアプリから直接コンピュータを作成することです。

1. **Settings → Droid Computers**に移動します。
2. **Create**をクリックします。
3. コンピュータに名前を付けます。

Factoryはクラウドコンピュータ（4 CPU、8GB RAM、6GBスワップ）を自動的にプロビジョニングします。プロビジョニングの進行状況は、以下のステップを通じてUIでライブ追跡されます：

1. **Creating computer** — クラウドコンピュータの割り当て
2. **Setting up user** — sudo権限を持つ`factory-user`アカウントの作成
3. **Configuring environment** — 環境設定、SSHキー、サービスファイルの書き込み
4. **Installing Droid** — Droidバイナリのダウンロードとインストール
5. **Starting services** — SSHとDroidデーモンサービスの起動

いずれかのステップが失敗すると、コンピュータのステータスが**Error**に変わり、失敗したステップがUIに表示されます。コンピュータを削除して再試行できます。

プロビジョニングが完了すると、コンピュータのステータスが**Active**に変わり、セッションで使用できるようになります。プラットフォーム管理のコンピュータは、アイドル時に自動一時停止し、新しいセッションがターゲットにしたときに自動再開します。また、プロビジョニングの詳細、リソースメトリクス、リモート更新フローも公開します。

## CLIコマンド

### 一般

| コマンド | 説明 |
|---------|-------------|
| `droid computer list` | すべてのコンピュータをリスト表示（名前、ID、ステータスを表示し、現在の機器をマーク） |
| `droid computer ssh <name>` | 任意のコンピュータへの対話的SSHセッションを開く |

### BYOMセットアップ

| コマンド | 説明 |
|---------|-------------|
| `droid computer register [name]` | 現在の機器をBYOMコンピュータとして登録 |
| `droid computer remove` | 現在の機器の登録を解除し、ローカル設定をクリーンアップ |

### SSHオプション

`ssh`サブコマンドは以下のフラグをサポートします：

- `--proxy` — stdioプロキシとして実行（`ProxyCommand`として使用）
- `--port <port>` — リモート機器のターゲットポート（デフォルト：22）
- `--debug` — 詳細な接続ログを有効化

## モニタリング

**Settings → Droid Computers**で任意のコンピュータをクリックして詳細ページを開きます。プラットフォーム管理のコンピュータの場合、詳細ページにはライブリソースメトリクスが表示されます：

- **CPU使用量** — 時系列の使用率パーセンテージ
- **メモリ使用量** — 使用済み vs. 総メモリ
- **ディスク使用量** — 使用済み vs. 総ディスク容量

詳細ページには現在の**デーモンバージョン**と接続ステータスも表示されます。可能なコンピュータステータスは：

- **Provisioning** — 初期セットアップが進行中
- **Active** — セッション準備完了
- **Error** — プロビジョニングまたは実行時エラー

## デーモンの更新

コンピュータ詳細ページには、実行中のデーモンバージョンと利用可能な最新バージョンが表示されます。更新が利用可能な場合、**Update**ボタンが表示されます。これをクリックするとリモート更新がトリガーされ、デーモンが新しいバイナリをダウンロードし、再起動して自動的に再接続します。

BYOM機器の場合、最新版をダウンロードしてデーモンプロセスを再起動することで、Droid CLIを手動で更新することもできます。

## Git認証情報

- **BYOM** — Droidは機器に既に設定されているgit認証情報を使用します。
- **マネージドコンピュータ** — FactoryとのGitHub連携がある場合、認証されたリポジトリアクセスのために認証情報が自動的に追加されます。新しいGitHub Appインストールを追加したり、リポジトリアクセスを変更したりすると、次回のセッション接続時に認証情報が自動的に更新されます。

## SSH & IDE統合

`droid computer ssh <name>`はデーモン経由でセキュアなWebSocketトンネルを確立します — 直接的なSSHポートの公開は不要です。Factoryは専用のEd25519 SSHキーペア（`~/.factory/.ssh/`に保存）を生成・管理し、個人のSSHキーとは分離されています。パブリックキーは各接続時にパスワードなし認証のために注入されます。

### VS Code Remote-SSH

`droid computer ssh`をVS Code Remote-SSHの`ProxyCommand`として使用できます：```
Host factory-mycomputer
  ProxyCommand droid computer ssh mycomputer --proxy
  User factory-user
  IdentityFile ~/.factory/.ssh/id_ed25519
  IdentitiesOnly yes
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```これにより、Droid Computer で完全な VS Code リモートセッションを開くことができます。

<Note>
プロキシモードは、コンピューターを解決する際に Host 名から `factory-` プレフィックスを自動的に削除します。例えば、`factory-mycomputer` という名前の Host エントリは、`mycomputer` という名前のコンピューターに接続されます。
</Note>

## セキュリティ

- **ファイアウォール** — Platform で管理されるコンピューターは iptables ルールを使用して、デーモンポートへのインバウンドトラフィックのみを制限し、その他すべてのポートはデフォルトでブロックされます。なお、`factory-user` はパスワードなしの sudo アクセス権を持ち、これらのルールを変更できます。ハードなネットワークレベルの境界が必要な場合は、インフラストラクチャレベルですべてのパブリックトラフィックをブロックするリレーモードを使用してください。
- **SSH の堅牢化** — root ログインとパスワード認証は無効になっています。
- **リレーモード** — BYOM コンピューターはすべてのトラフィックを Factory のリレーサービス経由でルーティングするため、パブリックポートは公開されません。Platform で管理されるコンピューターも、より厳格なネットワーク分離のために組織レベルでリレーモードを使用するよう設定できます。
- **Git 認証情報** — Platform で管理されるコンピューターでは、認証されたリポジトリアクセスのためにユーザーごとに自動的に設定されます。
