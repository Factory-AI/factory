Droid Control は Droid が*操作*するソフトウェアです：アプリの起動、コマンドの入力、ボタンのクリック、何が起こったかの記録、そして洗練されたビデオ証拠の生成を行います。Droid が開発し、Droid のために作られました。

## 始め方

<Tabs>
  <Tab title="UI">
    Droid セッションで `/plugins` を実行し、**Browse** タブに移動し、**droid-control** を見つけてインストールします。
  </Tab>
  <Tab title="CLI">
    ```bash
    droid plugin marketplace add https://github.com/Factory-AI/factory-plugins
    droid plugin install droid-control@factory-plugins --scope user
    ```</Tab>
</Tabs>

動画レンダリングでは、プラグインを追加した後にRemotion依存関係の一度だけのインストールが必要です：```bash
droid plugin list --scope user
cd <plugin-path>/remotion && npm install
```<Note>
使用ケースに応じたランタイムツール（tuistory、agent-browser、ffmpeg など）も必要です。使用ケース別のインストールコマンドについては、[前提条件](#prerequisites)を参照してください。
</Note>

## コマンド

Droid Controlは3つのスラッシュコマンドを追加します。それぞれが計画、実行、記録、レポートまでの全ワークフローをエンドツーエンドで処理します。

<Tabs>
  <Tab title="/verify">
    特定の動作要求をテストし、証拠とともに結果をレポートします。```
    /verify "ESC cancels streaming in bash mode"
    ```Droidはアプリを起動し、主張を試行して、実際に何が起こったかを報告します。証拠としてスクリーンショットとテキストスナップショットが含まれます。

<Tip>
droidは**調査者**として位置づけられており、支持者ではありません。主張が偽である場合、それは有効な発見です。偽造防止ルールにより、期待される結果に合わせて証拠を演出することは防止されています。
</Tip>
</Tab>
<Tab title="/qa-test">
ターミナルCLIやウェブ/Electronアプリに対して自動化されたQAを実行します。```
    /qa-test https://app.example.com -- login, create a project, invite a member
    ```Droidがブラウザ（またはターミナル）をフロー全体で操作し、各ステップをキャプチャし、注釈付きスクリーンショットと共に成功/失敗を報告します。
  </Tab>
  <Tab title="/demo">
    機能やPRのデモ動画を録画します。```
    /demo pr-1847
    ```Droidは、PRを読み取り、変更が機能することを証明するインタラクションをスクリプト化し、両方のブランチを並行して記録し、サイドバイサイドの比較動画を生成します。

さらなる磨きを加えるためのフラグを追加：```
    /demo pr-1847 -- showcase, keys
    ```| フラグ | 効果 |
    |------|--------|
    | `showcase` | 暖かい背景とフィルムグレインを持つシネマティックプリセット |
    | `keys` | ユーザーアクションを表示するキーストロークオーバーレイピル |
  </Tab>
</Tabs>

## `/demo`の仕組み

<Steps>
  <Step title="変更を理解する">
    PRの説明、差分、リンクされたチケットを取得します。何を証明する必要があるか、既存の動作と混同される可能性があることを特定します。
  </Step>
  <Step title="インタラクションを計画する">
    機能が動作することを視覚的に証明する一連のアクションをスクリプト化します。比較PRの場合、両方のブランチで同一のインタラクションを実行し、動作のみが異なるようにします。
  </Step>
  <Step title="両方のブランチをキャプチャする">
    ワーカーサブエージェントを使用して、ベースラインとカンディデートブランチで記録セッションを並行して起動します。
  </Step>
  <Step title="ビデオを合成する">
    Remotionを通じて、タイトルカード、ウィンドウクローム、キーストロークオーバーレイ、エフェクトを含む洗練されたビデオをレンダリングします。6つの視覚プリセットがシネマティックから実用的まで用意されています。
  </Step>
  <Step title="出力を検証する">
    配信前に最終ビデオが元のコミットメントと一致するかチェックします。
  </Step>
</Steps>

## 自動化ドライバー

Droid Controlは3つの自動化バックエンドをサポートしています。対象に基づいて自動的に適切なものが選択されます。

<CardGroup cols={3}>
  <Card title="tuistory" icon="terminal">
    **仮想PTY自動化。** ターミナル作業のデフォルト。asciinema記録と強制トゥルーカラー出力を持つPlaywright-styleCLI。
  </Card>
  <Card title="true-input" icon="keyboard">
    **実際のターミナルエミュレーター。** ヘッドレスWaylandコンポジター（Linux）、KVM/QEMU（Windows）、またはQEMUモニター（macOS）。実際のレンダリング証拠が必要な場合に使用。
  </Card>
  <Card title="agent-browser" icon="globe">
    **WebおよびElectronアプリ。** Chrome DevTools Protocolサポートを持つPlaywright-backedCLI。ページをナビゲートし、フォームを埋め、ボタンをクリックし、スクリーンショットをキャプチャします。
  </Card>
</CardGroup>

## ビデオレンダリング

デモとショーケースビデオは、Reactベースのビデオエンジンである[Remotion](https://www.remotion.dev/)でレンダリングされます。プラグインには22の視覚コンポーネントと6つのプリセットが含まれています。

<AccordionGroup>
  <Accordion title="視覚プリセット">
    | プリセット | 外観 | 最適な用途 |
    |--------|------|----------|
    | `factory` | 暖かい黒、信号機、アンバーグロー | 公式Factoryコンテンツ |
    | `factory-hero` | 同上 + グラデーション背景 | ランディングページ、ソーシャル |
    | `hero` | クールグラデーション、余裕のあるマージン | Factory以外のマーケティング |
    | `macos` | ダーク、クリーンフレーム | 汎用デモ |
    | `presentation` | 黒、余裕のあるマージン | スライドデッキ、プレゼンテーション |
    | `minimal` | ウィンドウバーなし、タイトマージン | ドキュメント埋め込み、インラインクリップ |
  </Accordion>
  <Accordion title="自動レイヤー（常に存在）">
    - 暖かい放射背景、浮遊パーティクル、フィルムグレインオーバーレイ、カラーグレーディング
    - モーションブラーのタイトルからコンテンツへの遷移
    - 信号機とガラスモーフィック境界を持つアニメーションウィンドウクローム
    - 自動スケールされたタイトル/サブタイトルテキスト
  </Accordion>
  <Accordion title="エフェクトレイヤー（合成時に選択）">
    - 特定の領域をハイライトするスポットライトオーバーレイ
    - 小さなテキストや詳細のための指向性ズーム
    - ユーザーアクションを表示するキーストロークピル
    - セクションヘッダーと遷移スイープ
  </Accordion>
</AccordionGroup>

## アーキテクチャ

プラグインは3つの層を持つコンポジションアーキテクチャを使用します：

- **オーケストレーター** -- 各リクエストを3つの独立したルックアップ（ターゲット、ステージ、アーティファクト）を通してルーティングし、ロードするスキルを決定します。
- **10個のアトムスキル** -- オンデマンドでロードされる自己完結型のバックグラウンド知識で、ドライバー、ターゲット、ステージ、ポリッシュに分かれています。
- **3つのコマンド** -- 引数をコミットメントに解析し、ハイブリッドハンドオフを通じてアトムに委任します。

すべてのワークフローは**キャプチャ → 合成 → 検証**を通ります。コマンドは*何を*生成するかを宣言し、アトムは*どのように*を所有します。スキルはハードコードされたパイプラインではなく明示的なハンドオフを通じて連鎖するため、droidは自然にフローに従います。

## 前提条件

使用ケースに必要なもののみインストールしてください。

<AccordionGroup>
  <Accordion title="ターミナルデモ（tuistory）">```bash
    npm install -g tuistory                                # virtual PTY driver
    pip install asciinema                                   # terminal recording
    cargo install --git https://github.com/asciinema/agg   # .cast → .gif converter
    sudo apt-get install -y ffmpeg                          # video processing
    ```</Accordion>
<Accordion title="Web/Electronの自動化（agent-browser）">```bash
    agent-browser install   # downloads Chromium
    ```</Accordion>
  <Accordion title="リアルターミナルエミュレーター（true-input）">
    | プラットフォーム | 必要なツール |
    |----------|----------------|
    | Linux/Wayland | `cage`、`wtype`、Wayland対応ターミナル |
    | Windows (KVM) | `libvirt`、`qemu`、SSH接続可能なKVM VM |
    | macOS (QEMU) | `qemu`、`socat`、SSH接続可能なmacOS VM |
  </Accordion>
  <Accordion title="動画合成（ショーケース）">
    Node.js >= 18、Chrome/Chromium、`ffmpeg`、`ffprobe`、`agg`が必要です。
  </Accordion>
</AccordionGroup>
