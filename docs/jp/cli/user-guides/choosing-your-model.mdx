## 3 · セッション中のモデル切り替え

- `/model`（または**Shift+Tab → Settings → Model**）を使用して、チャット履歴を失うことなく切り替え可能です。
- プロバイダーを変更する場合（例：AnthropicからOpenAI）、CLIはセッション記録をAnthropicとOpenAI形式間で変換します。変換は損失があります—プロバイダー固有のメタデータは削除されます—しかし実際には精度の低下は見られていません。
- 最適なコンテキスト継続性のため、自然なマイルストーンでモデルを切り替えてください：コミット後、PRがランドした後、または失敗したアプローチを放棄して計画をリセットする時。
- 頻繁に前後に切り替えると、アシスタントが再グラウンディングに1ターン費やすことが予想されます；切り替え時には最近の進捗の要約を検討してください。

## 4 · 推論努力設定

- **Opus 4.7**: Off / Low / Medium / High / **Max** (デフォルト: High)
- **Opus 4.6 / Opus 4.6 Fast**: Off / Low / Medium / High / **Max** (デフォルト: High)
- **Sonnet 4.6**: Off / Low / Medium / High / **Max** (デフォルト: High)
- **Opus 4.5 / Sonnet 4.5 / Haiku 4.5**: Off / Low / Medium / High (デフォルト: Off)
- **GPT-5.4**: None / Low / Medium / High / **Extra High** (デフォルト: Medium)
- **GPT-5.2**: Off / Low / Medium / High / **Extra High** (デフォルト: Low)
- **GPT-5.2-Codex**: None / Low / Medium / High / **Extra High** (デフォルト: Medium)
- **GPT-5.3-Codex**: None / Low / Medium / High / **Extra High** (デフォルト: Medium)
- **Gemini 3.1 Pro**: Low / Medium / High (デフォルト: High)
- **Gemini 3 Flash**: Minimal / Low / Medium / High (デフォルト: High)
- **Droid Core (GLM-5)**: Noneのみ (デフォルト: None；画像サポートなし)
- **Droid Core (GLM-5.1)**: Noneのみ (デフォルト: None；画像サポートなし)
- **Droid Core (Kimi K2.6)**: Off / High (デフォルト: High)
- **Droid Core (Kimi K2.5)**: Noneのみ (デフォルト: None)
- **Droid Core (MiniMax M2.7)**: Low / Medium / High (デフォルト: High)

推論努力は遅延とコストを増加させます—シンプルな作業では低く開始し、必要に応じてエスカレートしてください。**Max**はClaude Opus 4.7、Opus 4.6ファミリー（Opus 4.6とOpus 4.6 Fast）、およびSonnet 4.6で利用可能です。**Extra High**はGPT-5.4、GPT-5.2、GPT-5.2-Codex、およびGPT-5.3-Codexで利用可能です。

<Tip>`/model` → **Reasoning effort**、またはセッティングメニュー経由で推論努力を変更してください。</Tip>

## 5 · Bring Your Own Keys (BYOK)

Factoryは管理されたAnthropicとOpenAIアクセスが付属しています。独自のアカウントで実行したい場合、BYOKはオプトイン方式です—セットアップ手順、サポートされるプロバイダー、課金に関する詳細は[Bring Your Own Keys](/jp/cli/byok/overview)をご覧ください。

### オープンソースモデル

**Droid Core (GLM-5)**、**Droid Core (GLM-5.1)**、**Droid Core (Kimi K2.6)**、**Droid Core (Kimi K2.5)**、および**Droid Core (MiniMax M2.7)**はCLIで利用可能なオープンソース代替品です。以下の用途に役立ちます：

- 外部API呼び出しが許可されない**エアギャップ環境**
- 無制限ローカル推論が必要な**コスト重視プロジェクト**
- コードがインフラストラクチャを離れることができない**プライバシー要件**
- オープンソースモデル能力の**実験**

**注意：** GLM-5とGLM-5.1は画像添付をサポートしていません。Kimi K2.5、Kimi K2.6、およびMiniMax M2.7は画像をサポートします。Kimi K2.6にはOff/High推論切り替えが追加され、MiniMax M2.7（0.12×乗数で利用可能な最安価モデル）はLow/Medium/High推論をサポートします。画像ベースワークフローには、Claude、GPT、Kimi、またはMiniMax M2.7を使用してください。

オープンソースモデルを使用するには、ローカル推論サーバー（Ollamaなど）またはホストされたプロバイダーでBYOK経由で設定する必要があります。セットアップ手順については[BYOKドキュメント](/jp/cli/byok/overview)をご覧ください。

## 6 · 効果的な手法のメモを残す

- インパクトの高いワークフロー（例：仕様書生成 vs. クイック編集）と、どのモデル + 推論努力の組み合わせが最適に感じられるかを記録してください。
- モデルの回帰に気づいた場合はコミュニティやFactoryの担当者にお知らせください。迅速にベンチマークしてこのガイダンスを更新できます。
